Porque existe o Sensor Readout

Há muita informação sobre o próprio computador que costuma estar difícil de encontrar: escondida no Gestor de Dispositivos, espalhada por ferramentas antigas do Windows, bloqueada em aplicações pagas ou apresentada de forma pouco acessível. O Sensor Readout pretende juntar essa informação num só programa gratuito, portátil ou instalável, para que o utilizador não tenha de procurar em vários sítios.

Este projeto nasceu da frustração com o estado da acessibilidade na parte técnica da informática em Windows. A intenção é criar uma ferramenta que uma pessoa cega ou com baixa visão possa usar sem pedir a alguém para ler a janela, interpretar ícones ou navegar em painéis visuais pouco claros.

O objetivo é que técnicos de suporte, utilizadores avançados e clientes consigam descobrir mais sobre uma máquina, gerar relatórios úteis e resolver problemas com menos barreiras. O Sensor Readout tenta mostrar tudo o que consegue obter honestamente. Quando o Windows, o firmware, o controlador ou a biblioteca de sensores não expõem uma informação, a aplicação não inventa valores.

Espero que descubras tanto sobre o teu computador com esta aplicação como eu descobri sobre o meu, e que fiques agradavelmente surpreendido com o que encontrares.

O meu desejo é que esta aplicação se torne uma referência na sua área, e que pessoas cegas ou com baixa visão que trabalham em suporte técnico possam entregá-la aos seus clientes, pedir relatórios e resolver problemas com mais confiança.

Andre Louis, criador do Sensor Readout.

O que faz

O Sensor Readout lê sensores e informação de hardware a partir do Windows, LibreHardwareMonitor, Core Temp quando disponível, bases de dados incluídas e plug-ins opcionais. Mostra temperaturas, ventoinhas, estado SMART, rede, USB, áudio, ecrã, bateria, dispositivos, impressoras, firmware, CPU, memória, discos, volumes, detalhes de controladores e outros dados que o sistema expõe.

A janela principal tem uma lista de categorias à esquerda e uma árvore de leituras à direita. As leituras principais ficam limpas e diretas. Informação mais técnica fica atrás de Detalhes, acessível com Enter, pelo menu de contexto ou pelo comando Detalhes quando a leitura selecionada o suporta.

Também pode criar relatórios TXT e HTML, relatórios anonimizados, pacotes de suporte, comparações entre relatórios, histórico CSV de leituras, alarmes, perfis de ventoinha, curvas de ventoinha, hotkeys globais faladas e estado configurável da área de notificação.

O Sensor Readout não substitui ferramentas oficiais de firmware, atualização de BIOS ou software de fabricante. Ele lê e organiza informação; quando um plug-in oferece controlo de ventoinha, essa capacidade é explicitamente apresentada e depende do hardware suportar escrita segura.

Pré-requisitos

O Sensor Readout é uma aplicação Windows baseada em .NET Framework. Muitas leituras funcionam sem instalação extra, mas algumas funções de baixo nível precisam de componentes adicionais ou permissões elevadas.

  • Windows com .NET Framework compatível.
  • Execução como administrador para acesso completo a sensores de motherboard e controlo de ventoinhas em muitos sistemas.
  • PawnIO para acesso de baixo nível em hardware suportado.
  • Core Temp opcional para temperaturas e carga de CPU quando LibreHardwareMonitor não expõe esses dados.
  • Ferramentas de fabricante opcionais em portáteis, quando ajudam a expor dados específicos.

Instalador de pré-requisitos

O ficheiro Install-Prerequisites.cmd inicia o instalador PowerShell de pré-requisitos. Ele tenta usar ferramentas comuns como winget ou Chocolatey quando disponíveis e pode abrir as páginas oficiais quando uma instalação automática não é adequada.

O instalador de pré-requisitos é separado da aplicação principal. Não deve ser necessário executá-lo em todas as máquinas, mas pode ajudar quando faltam componentes necessários para sensores de baixo nível.

Comandos manuais de instalação

Se preferir instalar componentes manualmente, use as páginas oficiais dos respetivos projetos. Para Core Temp, use a página oficial do Core Temp. Para PawnIO, use a página do PawnIO.Setup. Instale apenas componentes que façam sentido para a sua máquina.

Primeira utilização

Ao iniciar, o Sensor Readout carrega as definições, escolhe um idioma com base no idioma do Windows quando possível, cria as pastas necessárias e começa a recolher leituras. Algumas leituras aparecem imediatamente; outras dependem de consultas mais lentas ou de hardware que demora a responder.

Se a aplicação estiver configurada para iniciar minimizada, recomenda-se fortemente configurar uma hotkey global para mostrar ou ocultar a janela. Sem essa tecla, pode ser difícil voltar à janela se o ícone da área de notificação estiver escondido pelo Windows.

O assistente inicial pode ajudar a definir a tecla de mostrar/ocultar e a tecla para falar o estado da área de notificação. Pode ser executado novamente no menu Ajuda.

Configuração de acessibilidade

O Sensor Readout foi pensado primeiro para teclado e leitor de ecrã. A maioria dos comandos tem aceleradores de menu, atalhos ou ambos. Os botões importantes têm nomes acessíveis e as leituras selecionadas podem ser revistas sem copiar primeiro.

  • Defina uma hotkey de mostrar/ocultar se usar a aplicação minimizada.
  • Defina uma hotkey para falar o estado da área de notificação se quiser ouvir leituras sem abrir a janela.
  • Use F3 para procurar numa categoria ou numa vista de detalhes.
  • Use F4 para rever o texto da leitura selecionada numa caixa de edição só de leitura.
  • Use Ctrl+C para copiar a leitura completa e Ctrl+Shift+C para copiar apenas o valor.
  • Quando uma leitura tem Detalhes, o leitor de ecrã anuncia essa informação no fim do texto da linha.

Opções de linha de comandos

As opções de linha de comandos servem para testes, suporte e automação. Algumas opções criam relatórios e fecham automaticamente. Outras controlam o arranque da aplicação.

  • --minimized: inicia minimizado na área de notificação.
  • --close: pede a uma instância em execução para fechar de forma normal.
  • --log-level: define o nível de registo.
  • --save-report-txt caminho: guarda um relatório TXT.
  • --save-report-html caminho: guarda um relatório HTML.
  • --save-anonymized-report-txt caminho: guarda um relatório TXT anonimizado.
  • --save-anonymized-report-html caminho: guarda um relatório HTML anonimizado.
  • --compare-reports antes depois saida: compara dois relatórios.
  • --diagnostics caminho: cria um ZIP de diagnóstico.
  • --community-stats-json caminho: gera silenciosamente o payload JSON das estatísticas anónimas da comunidade para teste local. Não faz upload.

Preferências

As Preferências estão em Opções > Preferências. A maioria das alterações é guardada imediatamente. Fechar a janela não é um botão de guardar; é apenas a forma normal de sair do diálogo.

Geral (Ctrl+1)

Inclui idioma, separador decimal, intervalo de atualização, atualizações automáticas, sons, mensagens faladas, comportamento minimizado e opções gerais de apresentação. O idioma pode ser alterado sem reiniciar a aplicação.

Arranque e Instalação (Ctrl+2)

Permite instalar a aplicação neste PC, criar atalho no ambiente de trabalho, configurar arranque com o Windows, remover a instalação local e escolher se a aplicação inicia minimizada. Quando a aplicação já está instalada no local esperado, o separador mostra opções de remoção em vez de continuar a oferecer a instalação.

Hotkeys (Ctrl+3)

Permite configurar hotkeys globais faladas. Cada perfil pode ter várias leituras, uma tecla, nomes falados personalizados e ordem própria. As alterações podem ser testadas imediatamente. Existem predefinições para estado do sistema, rede, atividade de disco, GPU, bateria e ventoinhas/temperaturas.

Perfis de ventoinha (Ctrl+4)

Perfis de ventoinha aplicam várias ações de ventoinha em conjunto, incluindo voltar ao modo automático. Podem ser ligados a hotkeys e sons, quando os controlos de ventoinha são suportados pela máquina.

Alarmes (Ctrl+5)

Alarmes monitorizam leituras e podem falar, tocar sons ou piscar o ícone da área de notificação. Existem predefinições para bateria fraca, Wi-Fi fraco ou desligado, CPU/GPU quente, pouca memória, pouco espaço em disco, saúde de disco baixa, atividade de disco alta, uptime longo e problemas de impressora.

Plug-ins (Ctrl+6)

Mostra plug-ins instalados e permite ativar ou desativar plug-ins opcionais. Os plug-ins ficam desativados por predefinição. Alterações aplicam-se depois de fechar as Preferências.

Itens ocultos (Ctrl+7)

Permite rever leituras ocultas, restaurá-las e reativar avisos que foram escondidos com opções do tipo não mostrar novamente.

Editor de idioma (Ctrl+8)

Edita ficheiros de idioma instalados sem abrir um editor externo. O inglês é o idioma de referência. Entradas em falta usam o inglês como alternativa. O editor suporta F3, lembra a última entrada editada e permite navegação por várias letras na lista.

Alarmes e sons

Alarmes podem usar comparações acima, abaixo, igual, diferente, ligado/desligado ou texto, conforme a leitura. Cada alarme tem tempo de espera para evitar repetição excessiva. Sons WAV podem ser colocados na pasta Sounds e escolhidos nas Preferências.

Os alarmes dependem da leitura existir. Se uma leitura só aparece numa máquina específica, um perfil importado pode precisar de ser reassociado nessa máquina.

Plug-ins

O sistema de plug-ins permite adicionar suporte específico para fabricantes sem tornar a aplicação principal mais arriscada. Os plug-ins incluídos podem procurar interfaces de portáteis Framework, Dell, Lenovo, MSI, Asus, HP, OMEN e Victus.

Para importar um plug-in, use Ficheiro > Importar plug-in ZIP. O plug-in é validado, copiado para a pasta certa e fica desativado até ser ativado manualmente.

Leitura dos sensores

As leituras vêm de várias fontes. LibreHardwareMonitor fornece muitos sensores de CPU, GPU, motherboard, armazenamento e ventoinhas. O Windows fornece informação de dispositivos, rede, bateria, impressoras, discos, volumes, drivers, firmware e WMI. Core Temp pode fornecer CPU quando necessário. Bases de dados incluídas dão nomes de fornecedores USB e MAC/OUI.

Algumas leituras são rápidas e atualizam frequentemente. Outras são cacheadas ou recolhidas em caminhos mais lentos para evitar uso excessivo de CPU e WMI. Isto é especialmente importante em portáteis lentos ou máquinas onde WMI fica caro.

Fluxo de controlo de ventoinhas

Controlo de ventoinhas só aparece quando o hardware expõe controlos escrevíveis. Muitas máquinas mostram RPM mas não permitem controlo. Antes de alterar ventoinhas, confirme que a leitura corresponde à ventoinha certa e que o sistema tem refrigeração segura.

  1. Execute como administrador.
  2. Confirme que as ventoinhas aparecem na categoria Ventoinhas.
  3. Abra Opções > Controlos de ventoinha.
  4. Ative Mostrar paradas se uma ventoinha controlável estiver escondida porque está em fan-stop.
  5. Teste alterações pequenas antes de criar curvas automáticas.

Curvas de ventoinha

Curvas ligam uma leitura de temperatura a um controlo de ventoinha. Uma curva define ponto baixo, ponto alto e ponto de emergência. O Sensor Readout aplica percentagens entre esses pontos quando a curva está ativa.

Curvas são poderosas e devem ser criadas com cuidado. Se uma curva for substituída temporariamente por controlo manual ou perfil, ela volta quando a ventoinha regressa ao modo automático/predefinido.

Perfis de ventoinha

Perfis aplicam várias ações de uma vez. Podem ser usados para voltar tudo ao automático, aplicar um modo silencioso, criar um modo de teste ou ligar uma configuração a uma hotkey. Como o suporte depende do hardware, uma configuração importada pode precisar de ajustes.

Exemplos de configuração

  • Hotkey de estado geral: CPU, memória usada, memória livre, uptime e GPU.
  • Hotkey de rede: estado, IP, velocidade, receção e envio.
  • Hotkey de disco: atividade, leitura, escrita e espaço livre dos discos importantes.
  • Alarme de bateria: abaixo de 20% com fala e som.
  • Alarme de Wi-Fi: desligado ou sinal baixo.
  • Perfil de ventoinha: voltar ao automático depois de testar uma percentagem manual.

Diagnósticos

O relatório de diagnóstico cria um ZIP com informação útil para suporte. Inclui resumo, relatórios, logs e dados relevantes para entender que fontes contribuíram para as leituras. Não deve substituir uma revisão cuidadosa antes de partilhar dados privados.

O comando de diagnóstico abre a pasta de relatórios quando termina e mostra o tempo gasto na linha de estado. Não deixa um diálogo escondido para trás quando a aplicação está minimizada.

Relatórios

Guardar um relatório

Ctrl+S guarda um relatório. HTML preserva melhor a estrutura e é normalmente a melhor escolha para partilhar. TXT é útil para leitura simples, mas alguns dados internos são quebrados em linhas para evitar linhas demasiado longas.

Estatísticas anónimas da comunidade

Ajuda > Partilhar estatísticas anónimas da comunidade cria um pequeno payload JSON separado dos relatórios anonimizados. A aplicação mostra exatamente o que será enviado antes do upload. Nada é enviado automaticamente.

O payload não inclui nome do computador, nome de utilizador, números de série, endereços MAC ou IP, caminhos, etiquetas de unidades, IDs de dispositivo, IDs PnP, detalhes brutos, programas instalados ou linhas completas de relatório. Uma ID aleatória local é guardada no computador, e apenas o hash SHA-256 dessa ID é enviado para permitir atualizar a entrada agregada da mesma máquina.

Depois de enviar, a aplicação abre a página pública de estatísticas: https://onj.me/srstats/. A página também pode ser visitada sem enviar estatísticas.

Abrir o relatório de outra pessoa

Relatórios HTML, TXT e alguns ZIPs de diagnóstico podem ser abertos no Sensor Readout. Quando um relatório está aberto, a janela indica que está em modo de relatório. Hotkeys e estado da área de notificação referem-se ao relatório carregado, não à máquina ao vivo.

Comparar relatórios

A comparação mostra leituras adicionadas, removidas, alteradas e diferenças de detalhes. É útil para comparar antes/depois de uma atualização, duas máquinas diferentes ou o mesmo computador com configurações diferentes.

Relatórios de suporte

O relatório de suporte prepara um ZIP e dá um fluxo orientado para copiar o caminho, abrir a pasta ou abrir a página de issues. Isto reduz erros ao enviar dados para análise.

Histórico de leituras

O histórico grava leituras selecionadas num CSV temporário em Logs. É útil para observar tendências curtas, como temperatura, carga, rede ou disco durante uma tarefa.

Transferência de definições

Exportar definições cria um ficheiro JSON com perfis, hotkeys, alarmes e outras escolhas. Importar permite escolher que partes trazer. Leituras genéricas como CPU, memória e uptime podem funcionar diretamente; leituras ligadas a hardware específico podem precisar de nova associação.

Também existe uma opção para empacotar uma cópia instalada com configuração, sons e personalizações para usar noutro local. Reveja sempre a cópia antes de a partilhar.

Ficheiros

  • Config: definições partilhadas e específicas da máquina.
  • Logs: logs de execução e histórico CSV.
  • Reports: relatórios guardados.
  • Docs: manuais HTML incluídos.
  • Langs: ficheiros de idioma.
  • Data: bases de dados de fornecedores USB/MAC e hashes de idiomas incluídos.
  • Plug-Ins: plug-ins opcionais.
  • Backups: backups criados durante atualizações quando necessário.

Notas

Suporte de ventoinhas depende de LibreHardwareMonitor, PawnIO, chip de sensores da motherboard, firmware e permissões. GPUs e discos podem aparecer sem elevação, mas motherboard e ventoinhas frequentemente precisam de administrador.

Se a temperatura ou carga de CPU estiver em falta, instalar e executar Core Temp pode ajudar. O Core Temp é opcional e usado apenas como fallback.

Se controlos de ventoinha não aparecerem, abra Opções > Controlos de ventoinha e ative Mostrar paradas. Algumas placas reportam cabeçalhos controláveis como parados ou indefinidos até começarem a rodar.

Plug-ins de marcas de portáteis

Utilizadores de Framework, Dell, Lenovo, MSI, Asus, HP, OMEN e Victus podem obter melhor visibilidade de ventoinhas ou temperaturas ativando o plug-in correspondente. Estes plug-ins não garantem dados em todos os modelos.

Ferramentas do fabricante também podem ajudar: Dell Command | Monitor, Lenovo Vantage, Lenovo System Update, MSI Center, G-Helper, HP Support Assistant e OMEN Gaming Hub. São ferramentas externas; instale apenas o que fizer sentido para a sua máquina.

O Sensor Readout usa ou inclui componentes destes projetos:

  • LibreHardwareMonitor e LibreHardwareMonitor no GitHub, para acesso a sensores de hardware.
  • PawnIO e PawnIO.Setup, para acesso de baixo nível.
  • Newtonsoft.Json, para ficheiros de configuração.
  • HidSharp, usado por LibreHardwareMonitor.
  • DiskInfoToolkit, para dados de armazenamento.
  • RAMSPDToolkit, para dados SPD de memória.
  • BlackSharp.Core, usado por dependências de LibreHardwareMonitor.
  • Tolk screen reader library, para fala opcional por leitor de ecrã.
  • Framework Control, usado pela API local opcional.
  • G-Helper, cuja investigação ACPI GPL é usada no plug-in Asus opcional.
  • usb.ids, usado para nomes de fornecedores e produtos USB.
  • MAC Address Vendor Database, usado para fornecedores MAC/OUI.
  • Microsoft .NET Framework e bibliotecas de suporte.
