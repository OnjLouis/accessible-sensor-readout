Requisitos

  • Windows 10 o Windows 11, 64 bits.
  • Microsoft .NET Framework 4.8 o posterior.
  • Controlador PawnIO instalado para sensores de placa base y control de ventiladores.
  • Derechos de administrador para leer muchos sensores de placa base o controlar ventiladores.

Diagn횄쨀sticos

Use Ayuda > Ejecutar diagnosticos... o pulse Alt+F1 cuando necesite reunir informacion de soporte. Sensor Readout crea un archivo ZIP en Reports con el nombre del equipo en el nombre del archivo, abre la carpeta al terminar y elimina la carpeta temporal despues de crear el ZIP.

La prueba comprueba brevemente los controles de ventilador escribibles al 100% y luego restaura cada ventilador a su estado anterior manual, automatico/predeterminado o de curva de ventilador.

Los diagnosticos pueden decir el progreso y reproducir sonidos de inicio/final. Configure esto en Opciones > Preferencias > Inicio. Los diagnosticos por linea de comandos usan las mismas preferencias salvo que use --diagnostics-quiet, --no-diagnostics-speech o --no-diagnostics-sounds.

Para pruebas desatendidas, ejecute Sensor Readout.exe --diagnostics [ruta]. Si [ruta] es una carpeta, el ZIP se crea alli. Si no se indica ruta, Sensor Readout crea el ZIP en Reports.

Informes

Los informes sirven para dos situaciones: guardar una instantanea de su propio equipo o enviar esa instantanea a otra persona para que pueda verla dentro de Sensor Readout.

Guardar un informe

Pulse Ctrl+S o use Archivo > Guardar informe....

El dialogo de guardado ofrece informes TXT y HTML. TXT agrupa por tipo de lectura y dispositivo, para que los nombres largos de hardware se escriban una vez como encabezados. HTML usa tablas con sensor, valor y fuente.

El primer guardado crea la carpeta Reports si no existe. Para compartir un informe, envie el archivo .html o .txt guardado. HTML suele ser mas comodo de leer en un navegador; TXT es util para pegar en mensajes. Si alguien necesita un paquete de soporte mas completo, use Ayuda > Ejecutar diagnosticos...; los diagnosticos incluyen ambos formatos, registros y un resumen en un ZIP.

Abrir el informe de otro equipo

Pulse Ctrl+O o use Archivo > Abrir informe... para cargar un informe TXT o HTML guardado por Sensor Readout.

El informe se abre como una instantanea estatica en la vista normal de categorias y arbol. Asi puede inspeccionar el equipo de otra persona como si fuera su vista actual. El modo de informe estatico no actualiza valores en vivo, no ejecuta alarmas y no controla hardware. Solo muestra los datos guardados en el informe.

Los detalles y comandos de copia siguen funcionando cuando el informe contiene los campos necesarios. Esto ayuda cuando alguien envia datos USB, audio, pantalla, red, ventiladores, bateria o SMART y usted quiere revisar filas concretas sin leer todo el archivo manualmente.

Use Archivo > Volver a lecturas en vivo o Ctrl+R para volver al equipo actual. Los informes guardados por Sensor Readout 2.0 o posterior incluyen una instantanea legible por la app para una mejor importacion. Los informes TXT y HTML antiguos se abren lo mejor posible.

Transferencia de configuracion

Use Archivo > Exportar configuracion y perfiles... o Ctrl+E para crear un paquete de configuracion. Sensor Readout muestra primero una lista de comprobacion, para que pueda elegir exactamente que incluir. Use Archivo > Importar configuracion y perfiles... o Ctrl+Mayus+I para recuperar configuracion. La importacion muestra el mismo tipo de lista, limitada a las secciones encontradas en el paquete seleccionado. Las lecturas portables como memoria, CPU, tiempo activo del sistema y muchos valores de resumen se emparejan cuando es posible. Los elementos especificos del hardware se importan con cuidado: las teclas globales quedan vacias, las acciones de perfiles de ventilador no se vinculan, las curvas de ventilador quedan desactivadas hasta elegir un control de ventilador, y las alarmas quedan desactivadas para revision.

Notas

La compatibilidad de ventiladores depende de LibreHardwareMonitor, PawnIO, el chip sensor y la BIOS. Algunos sistemas dan lecturas pero no controles escribibles. Para Super I/O de placa base normalmente se requieren derechos de administrador.

Si faltan la temperatura o la carga de CPU, puede ayudar instalar y ejecutar Core Temp. Sensor Readout puede leer los datos de memoria compartida de Core Temp cuando Core Temp ya esta en ejecuci횄쨀n. Core Temp es opcional y solo se usa como fuente alternativa para lecturas de CPU. Use Ayuda > Instalar soporte Core Temp... para abrir la p횄징gina oficial de Core Temp o iniciar la instalacion con winget o Chocolatey cuando esas herramientas esten disponibles.

Si no aparecen controles de ventilador, abra Opciones > Controles de ventilador... y active Mostrar detenidos. Algunas placas informan cabeceras controlables como detenidas o indefinidas hasta que empiezan a girar; esta opci횄쨀n las muestra para probar.

Plug-ins de marca de portatil

Los usuarios de portatiles Framework, Dell, Asus, HP, OMEN y Victus pueden obtener mejor visibilidad de temperaturas o ventiladores activando el plug-in de hardware correspondiente en Opciones > Preferencias > Plug-Ins. Los plug-ins estan desactivados hasta que los active, y los cambios se aplican al cerrar Preferencias.

Los usuarios de Framework que quieran RPM de ventilador y temperaturas especificas del modelo deben actualizar primero la BIOS desde Framework BIOS and Drivers Downloads, instalar y ejecutar Framework Control. Si Framework Control muestra controles de ventilador y temperatura en su interfaz de navegador, active el plug-in Framework Laptop en Sensor Readout.

Los plug-ins Dell, Asus, HP, OMEN y Victus no garantizan lecturas de ventilador en todos los modelos, pero conviene probarlos antes de enviar diagnosticos o asumir que el portatil no expone mas datos de sensores.

Las herramientas opcionales del fabricante tambien pueden ayudar a exponer o comprobar datos especificos del portatil. Los usuarios de Dell pueden probar Dell Command | Monitor. Los usuarios de Asus pueden probar G-Helper. Los usuarios de HP, OMEN y Victus pueden probar HP Support Assistant y OMEN Gaming Hub. Estas herramientas no forman parte de Sensor Readout; use las paginas del fabricante o del proyecto e instale solo lo que tenga sentido para su equipo.

Sensor Readout solo lee estas rutas de soporte opcionales salvo que un plug-in indique otra cosa; no actualiza firmware ni reemplaza las herramientas del fabricante.
