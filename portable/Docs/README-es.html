Sensor Readout

Version actual: 1.4.1.

Sensor Readout es una utilidad de Windows para leer sensores de hardware y controlar ventiladores compatibles con una interfaz pensada primero para teclado y lectores de pantalla.

A la izquierda muestra categorias principales, a la derecha una vista de arbol con lecturas agrupadas por dispositivo o finalidad, y los comandos comunes estan en una barra de menus normal.

Sensor Readout se puede encontrar en GitHub.

Que hace

  • Lee temperaturas, RPM de ventiladores, salud y capacidad de almacenamiento, y contadores de hardware seleccionados.
  • Muestra una pagina Rendimiento/Vista general con tiempo activo, BIOS, GPU, uso de CPU, uso de memoria y actividad de lectura/escritura de unidades.
  • Abre la ventana principal de inmediato mientras la primera actualizacion de sensores continua en segundo plano.
  • Muestra una categoria Red con estado del adaptador, direcciones IP, velocidad de enlace, tasas de envio/recepcion y trafico total.
  • Enlaza la pagina del proyecto desde README, menu Ayuda y dialogo Acerca de.
  • Usa bibliotecas LibreHardwareMonitor incluidas para acceder a sensores.
  • Usa el controlador PawnIO para sensores de placa base y control de ventiladores de bajo nivel cuando el hardware lo permite.
  • Permite poner nombres faciles a cabeceras de ventilador.
  • Oculta por defecto cabeceras de ventilador detenidas o sin usar.
  • Aplica porcentajes manuales a un ventilador seleccionado o a todos los ventiladores visibles.
  • Devuelve un ventilador o todos a control automatico/predeterminado.
  • Guarda informes TXT o HTML.
  • Permite configurar la actualizacion automatica y usa 5 segundos como valor predeterminado en configuraciones nuevas.
  • Puede ejecutarse al iniciar Windows e iniciar minimizado en el area de notificacion.
  • Usa un archivo de configuracion por equipo en Config, por ejemplo Desktop.json o Laptop.json, para que una copia en Dropbox o USB funcione en varios equipos sin sobrescribir ajustes.
  • Guarda los cambios de preferencias al momento para proteger trabajo de hotkeys, bandeja, elementos ocultos y opciones similares.
  • Escribe diagnostico en Logs\<ComputerName>.log cuando el registro esta activado.
  • Puede mostrar temperaturas en Celsius, Fahrenheit, Celsius luego Fahrenheit o Fahrenheit luego Celsius.
  • Convierte contadores SMART de Samsung a valores GB/TB legibles cuando LibreHardwareMonitor expone unidades sin procesar.
  • Admite hotkeys globales opcionales para mostrar/ocultar y leer el estado del area de notificacion.
  • Puede hablar el estado del area de notificacion mediante NVDA con las DLL de NVDA Controller Client de 64 bits incluidas.
  • Admite archivos de idioma editables en la carpeta Langs.
  • El registro esta desactivado por defecto y se puede activar para solucionar problemas.
  • Puede mostrar lecturas seleccionadas en el tooltip del area de notificacion.

Requisitos

  • Windows 10 o Windows 11, 64 bits.
  • Microsoft .NET Framework 4.8 o posterior.
  • Controlador PawnIO instalado para sensores de placa base y control de ventiladores.
  • Derechos de administrador para leer muchos sensores de placa base o controlar ventiladores.

No hace falta ejecutar LibreHardwareMonitor como aplicacion separada porque esta carpeta incluye las bibliotecas necesarias. Instalar la aplicacion independiente puede ser util para diagnostico.

Comandos de instalacion con winget

Abra Windows Terminal o PowerShell como administrador y ejecute:

winget install --id Microsoft.DotNet.Framework.Runtime -e --source winget
winget install --id namazso.PawnIO -e --source winget

Herramienta opcional de diagnostico:

winget install --id LibreHardwareMonitor.LibreHardwareMonitor -e --source winget

Tambien puede ejecutar Install-Prerequisites.cmd. Este llama a Install-Prerequisites.ps1, pide administrador si hace falta, comprueba winget e instala .NET Framework Runtime y PawnIO. Para instalar tambien LibreHardwareMonitor independiente:

.\Install-Prerequisites.ps1 -IncludeLibreHardwareMonitor

Primeros pasos

1. Inicie Sensor Readout.exe.

2. Acepte el aviso de administrador de Windows.

3. Si falta PawnIO, Sensor Readout ofrece ejecutar el instalador de requisitos.

4. Pulse F5 si los valores aun se estan cargando.

5. Abra Opciones > Controles de ventilador... o pulse Ctrl+L para ajustar ventiladores.

Si faltan ventiladores o controles de placa base, compruebe PawnIO y que la app se haya iniciado como administrador. El instalador se puede volver a ejecutar desde Ayuda > Instalar requisitos....

Preferencias

Use Opciones > Preferencias o Ctrl+,.

  • Actualizacion automatica activada o desactivada.
  • Actualizar mientras Sensor Readout tiene foco.
  • Intervalo de actualizacion.
  • Icono del area de notificacion.
  • Inicio con Windows e inicio minimizado.
  • Hasta cuatro lecturas en el tooltip del area de notificacion.
  • Lecturas y grupos ocultos.
  • Nivel de registro: Desactivado, Error, Normal o Debug.
  • Si la voz incluye nombres de dispositivos antes de las lecturas seleccionadas.

Si el estado del area de notificacion esta activado, minimizar oculta Sensor Readout de la barra de tareas y Alt+Tab. Abralo desde el icono. Alt+F4 sale de la app.

La opcion de inicio crea o elimina Sensor Readout.lnk en la carpeta Inicio del usuario. Si se activa, tambien se activa inicio minimizado.

Las lecturas del area de notificacion se mueven de una lista disponible a una lista de orden. Windows muestra tooltips cortos, por eso el limite es cuatro. Una lectura aparece en una sola lista. Use botones o Ctrl+flechas. Los elementos muestran dispositivo, nombre y categoria, por ejemplo Ethernet - Rx: Red.

Leer sensores

El panel de lecturas es una vista de arbol. Temperaturas, Ventiladores, SMART, Rendimiento y Red agrupan por dispositivo o finalidad y despues por valor.

La lista izquierda solo cambia la vista; no activa ni desactiva dispositivos. Sensor Readout inicia en Temperaturas. Rendimiento agrupa CPU, memoria y actividad de unidades. Red muestra estado, IP, velocidad, tasas y contadores.

Las lecturas de porcentaje tambien se exponen en la barra de progreso inferior para que los lectores de pantalla usen su feedback habitual. Use Edicion, tecla Aplicaciones o clic derecho para copiar, renombrar un ventilador u ocultar valores.

Alarmas y sonidos

Preferencias > Alarmas permite crear alarmas de lecturas. Elija una lectura, Mayor o igual, Menor o igual, o Igual, y despues el umbral y la espera. Cada alarma puede hablar con NVDA, reproducir un WAV, o ambas cosas.

Preferencias > Inicio incluye sonidos opcionales de inicio y salida. Sensor Readout carga WAV desde la carpeta Sounds. Los sonidos incluidos usan nombres neutros como SR01.wav, SR02.wav, etc.; el usuario puede agregar sus propios .wav a la misma carpeta.

Flujo de control de ventiladores

Abra Opciones > Controles de ventilador... o pulse Ctrl+L.

1. Elija un control. 2. Opcional: escriba un nombre y pulse Alt+L. 3. Escriba un porcentaje de 0 a 100. 4. Pulse Alt+M. 5. Alt+A vuelve el seleccionado a automatico. 6. Alt+R vuelve todos los visibles a automatico.

Los botones para todos los ventiladores solo afectan controles visibles. Las cabeceras detenidas o vacias estan ocultas hasta activar Mostrar detenidos. Los nombres se guardan en Config y solo cambian como aparecen en Sensor Readout.

Informes

Pulse Ctrl+S o use Archivo > Guardar informe.... TXT agrupa por tipo y dispositivo. HTML usa tablas con sensor, valor y fuente.

Archivos

Build portable: Sensor Readout.exe.

  • Config\<ComputerName>.json: preferencias de actualizacion, area de notificacion, elementos ocultos, nombres de ventilador y registro.
  • Logs\<ComputerName>.log: diagnostico opcional, acciones de ventilador, hotkeys y mensajes de NVDA.

Archivos de idioma

  • Ponga archivos .txt o .ini en Langs.
  • En el primer inicio se elige el idioma de Windows si existe, o ingles.
  • Use lineas key=value; # y ; son comentarios.
  • language.name es el nombre mostrado.
  • manual.file=README-xx.html indica el manual para F1. Solo nombre de archivo, sin ruta. Se busca en Docs y si falla se usa ingles.
  • number.decimalSeparator=, define el separador decimal predeterminado del idioma.
  • La carpeta se revisa cada 15 segundos aproximadamente.
  • Langs\English.txt es el idioma principal y plantilla para idiomas nuevos.
  • El separador decimal tambien se puede cambiar en Preferencias.

La voz opcional de NVDA usa nvdaControllerClient.dll y nvdaControllerClient64.dll si NVDA esta en ejecucion. Al iniciar minimizado puede hablar un mensaje configurable. El editor de idioma permite editar sin abrir otro editor.

Notas

La compatibilidad de ventiladores depende de LibreHardwareMonitor, PawnIO, el chip sensor y la BIOS. Algunos sistemas dan lecturas pero no controles escribibles. Para Super I/O de placa base normalmente se requieren derechos de administrador.

Si faltan la temperatura o la carga de CPU, puede ayudar instalar y ejecutar Core Temp. Sensor Readout puede leer los datos de memoria compartida de Core Temp cuando Core Temp ya esta en ejecucion. Core Temp es opcional y solo se usa como fuente alternativa para lecturas de CPU.

Si no aparecen controles de ventilador, abra Opciones > Controles de ventilador... y active Mostrar detenidos. Algunas placas informan cabeceras controlables como detenidas o indefinidas hasta que empiezan a girar; esta opcion las muestra para probar.
