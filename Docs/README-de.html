Sensor Readout

Aktuelle Version: 1.6.1.

Sensor Readout ist ein Windows-Werkzeug zum Lesen von Hardware-Sensoren und zum Steuern unterstuetzter Luefter. Die Oberflaeche ist zuerst fuer Tastatur und Screenreader gebaut.

Links stehen die Hauptbereiche, rechts eine Baumansicht mit Messwerten nach Geraet oder Zweck. Die wichtigsten Befehle liegen in einer normalen Menueleiste.

Sensor Readout finden Sie auf GitHub.

Was es macht

  • Liest Temperaturen, Luefterdrehzahlen, Speichergesundheit, Speicherkapazitaet und ausgewaehlte Hardware-Zaehler.
  • Zeigt eine Leistungs-/Uebersichtsseite fuer Laufzeit, BIOS, GPU, CPU-Nutzung, Speichernutzung und Laufwerksaktivitaet.
  • Oeffnet die Hauptoberflaeche sofort, waehrend die erste Aktualisierung im Hintergrund laeuft.
  • Zeigt Netzwerkadapter mit Status, IP-Adressen, Verbindungsgeschwindigkeit, Sende-/Empfangsraten und Gesamtverkehr.
  • Zeigt MAC-Adressen und OUI-Hersteller von Netzwerkadaptern, wenn die mitgelieferte OUI-Datenbank den Praefix kennt.
  • Zeigt eine USB-Kategorie fuer verbundene Geraete, Hubs, Controller, Verbindungsgeschwindigkeit, Stromaufnahme wo Windows sie bereitstellt, Laufwerksbuchstaben, Sicher-entfernbar-Status und kopierbare Detailfelder.
  • Zeigt Laptop-Akku-Ladung, Status, Kapazitaet, Zustand, Ladezyklen, Spannung und Leistungsrate, wenn Windows sie bereitstellt.
  • Unterstuetzt einfache Luefterkurven, die eine Lueftersteuerung aus einer gewaehlten Temperatur ableiten.
  • Unterstuetzt Luefterprofile, die mehrere Lueftersteuerungen auf einmal setzen koennen, optional mit globalem Hotkey und optionaler Zurueck-auf-Automatik-Umschaltung.
  • Verlinkt die Projektseite aus README, Hilfemenue und Info-Dialog.
  • Verwendet mitgelieferte LibreHardwareMonitor-Bibliotheken fuer Sensorzugriff.
  • Verwendet PawnIO fuer Low-Level-Mainboard-Sensoren und Lueftersteuerung, sofern die Hardware dies unterstuetzt.
  • Erlaubt freundliche Namen fuer Luefteranschluesse.
  • Blendet gestoppte oder unbestueckte Mainboard-Luefter standardmaessig aus.
  • Setzt manuelle Luefterprozente fuer einen ausgewaehlten Luefter oder alle sichtbaren Luefter.
  • Setzt einen Luefter oder alle Luefter wieder auf automatische Standardsteuerung.
  • Speichert Sensorberichte als TXT oder HTML.
  • Unterstuetzt konfigurierbare automatische Aktualisierung und startet neue Konfigurationen mit 5 Sekunden Intervall.
  • Kann beim Windows-Start ausgefuehrt und minimiert in den Infobereich gestartet werden.
  • Verwendet Config\Shared.json fuer portable Einstellungen, die zwischen Computern mitwandern sollen, waehrend hardwarespezifische Einrichtung in Config\<ComputerName>.json bleibt.
  • Speichert Einstellungsanderungen sofort, damit Hotkeys, Infobereich, ausgeblendete Elemente und aehnliche Arbeit besser erhalten bleiben.
  • Schreibt Diagnoseprotokolle in Logs\<ComputerName>.log, wenn Protokollierung aktiviert ist.
  • Kann Temperaturen in Celsius, Fahrenheit, Celsius dann Fahrenheit oder Fahrenheit dann Celsius anzeigen.
  • Unterstuetzt globale Hotkeys zum Anzeigen/Ausblenden und zum Vorlesen des Infobereich-Status.
  • Kann den Infobereich-Status ueber Screenreader sprechen, wenn die mitgelieferten 64-Bit-Screenreader-Controller-DLLs verfuegbar sind.
  • Unterstuetzt benutzerbearbeitbare Sprachdateien im Ordner Langs.
  • Protokollierung ist standardmaessig aus und kann zur Fehlersuche aktiviert werden.
  • Kann ausgewaehlte Messwerte im Tooltip des Infobereichs anzeigen.

Voraussetzungen

  • Windows 10 oder Windows 11, 64-Bit.
  • Microsoft .NET Framework 4.8 oder neuer.
  • PawnIO-Treiber fuer Mainboard-Sensoren und Lueftersteuerung.
  • Administratorrechte zum Lesen vieler Mainboard-Sensoren oder zum Steuern von Lueftern.

LibreHardwareMonitor muss nicht als eigenes Programm laufen, weil dieser Ordner die benoetigten Bibliotheken mitliefert. Die eigenstaendige App kann zur Fehlersuche trotzdem nuetzlich sein.

Winget-Installationsbefehle

Oeffnen Sie Windows Terminal oder PowerShell als Administrator und fuehren Sie aus:

winget install --id Microsoft.DotNet.Framework.Runtime -e --source winget
winget install --id namazso.PawnIO -e --source winget

Optionales Werkzeug zur Fehlersuche:

winget install --id LibreHardwareMonitor.LibreHardwareMonitor -e --source winget

Install-Prerequisites.cmd ruft Install-Prerequisites.ps1 auf, fragt bei Bedarf nach Administratorrechten, prueft winget und installiert .NET Framework Runtime sowie PawnIO. Fuer die eigenstaendige LibreHardwareMonitor-App verwenden Sie:

.\Install-Prerequisites.ps1 -IncludeLibreHardwareMonitor

Erste Schritte

1. Starten Sie Sensor Readout.exe.

2. Bestaetigen Sie die Windows-Administratorabfrage.

3. Wenn PawnIO fehlt, bietet Sensor Readout an, den Prerequisite-Installer zu starten.

4. Druecken Sie F5, falls Werte noch geladen werden.

5. Oeffnen Sie Optionen > Lueftersteuerung... oder druecken Sie Strg+L, wenn Sie Luefterwerte anpassen moechten.

Wenn Mainboard-Luefter oder Steuerungen fehlen, pruefen Sie PawnIO und Administratorstart. Den Installer koennen Sie spaeter ueber Hilfe > Voraussetzungen installieren... erneut starten.

Einstellungen

Verwenden Sie Optionen > Einstellungen oder Strg+,.

  • Automatische Aktualisierung ein/aus.
  • Aktualisierung bei Tastaturfokus.
  • Aktualisierungsintervall.
  • Infobereich-Symbol.
  • Windows-Start und minimierter Start.
  • Bis zu vier Infobereich-Messwerte mit Reihenfolge.
  • Ausgeblendete Messwerte und Gruppen.
  • Protokollstufe: Aus, Fehler, Normal oder Debug.
  • Ob gesprochenes Feedback Geraetenamen vor ausgewaehlten Messwerten enthaelt.

Wenn Infobereich-Status aktiv ist, blendet Minimieren Sensor Readout aus Taskleiste und Alt+Tab aus. Oeffnen Sie es wieder ueber das Infobereich-Symbol. Alt+F4 beendet die App.

Die Startoption erstellt oder entfernt Sensor Readout.lnk im Startup-Ordner. Wenn Windows-Start aktiv ist, wird minimierter Start eingeschaltet.

Infobereich-Messwerte werden aus einer verfuegbaren Liste in eine Reihenfolge verschoben. Maximal vier Messwerte passen sinnvoll in Windows-Tooltips. Ein Messwert erscheint nur in einer Liste. Nutzen Sie Schaltflaechen oder Strg-Pfeile. Eintraege stehen als Geraet, Messwert und Kategorie, z. B. Ethernet - Rx: Netzwerk.

Der Hotkeys-Tab kann gesprochene Hotkey-Profile aus einer anderen Computer-Konfiguration importieren. Importierte Profile behalten ihre alten Tastenkombinationen nicht, damit sie keine Tasten auf diesem Computer belegen. Messwerte werden nur behalten, wenn Sensor Readout sie auf dem aktuellen Computer sicher zuordnen kann.

Sensoren lesen

Der Messwertbereich ist eine Baumansicht. Temperaturen, Luefter, SMART, Leistung und Netzwerk gruppieren nach Geraet oder Zweck und danach nach einzelnen Werten.

Die linke Liste wechselt nur die Ansicht; sie aktiviert oder deaktiviert keine Geraete. Sensor Readout startet auf Temperaturen. Leistung fasst CPU, Speicher und Laufwerksaktivitaet zusammen. Netzwerk zeigt Adapterstatus, IP-Adresse, Linkgeschwindigkeit, Raten und Gesamtzaehler.

Prozentwerte werden auch im Fortschrittsbalken unter der Baumansicht ausgegeben. Das hilft visuell und Screenreadern. Ueber Bearbeiten, Anwendungstaste oder Rechtsklick koennen Sie kopieren, Details oeffnen, Luefter umbenennen oder Werte ausblenden.

Alarme und Klaenge

Einstellungen > Alarme erstellt Messwert-Alarme. Waehlen Sie einen Messwert, Groesser oder gleich, Kleiner oder gleich oder Gleich, dann Grenzwert und Pause. Jeder Alarm kann per Screenreader sprechen, eine WAV-Datei abspielen oder beides.

Einstellungen > Start enthaelt optionale Start- und Beenden-Klaenge. Sensor Readout laedt WAV-Dateien aus dem Ordner Sounds. Die mitgelieferten Klaenge heissen neutral SR01.wav, SR02.wav usw.; eigene .wav-Dateien koennen in denselben Ordner gelegt werden.

Plug-Ins

Einstellungen > Plug-Ins listet installierte Hardware-Plug-Ins und beschreibt ihren Zweck. Plug-Ins sind vertrauenswuerdiger Code; aktivieren Sie nur Plug-Ins aus Quellen, denen Sie vertrauen.

Installierte Plug-Ins sind standardmaessig deaktiviert. Der ZIP-Import kopiert ein Plug-In in den Ordner Plug-Ins, aktiviert es aber nicht automatisch.

Das mitgelieferte Framework-Laptop-Plug-In ist optional und standardmaessig deaktiviert. Es kann Framework-spezifische Temperatur- und Luefter-RPM-Zeilen ergaenzen, wenn Framework Control installiert ist und laeuft.

Entwicklerhinweise stehen im GitHub-Repository unter Docs\\Plug-In-development.md.

Lueftersteuerung

Oeffnen Sie Optionen > Lueftersteuerung... oder Strg+L.

1. Lueftersteuerung waehlen. 2. Optional Namen eingeben und Alt+L. 3. Prozentwert 0 bis 100 eingeben. 4. Alt+M anwenden. 5. Alt+A setzt die Auswahl auf automatisch. 6. Alt+R setzt alle sichtbaren Steuerungen auf automatisch.

Alle-Luefter-Schaltflaechen wirken nur auf sichtbare Steuerungen. Gestoppte oder unbestueckte Mainboard-Anschluesse sind verborgen, bis Gestoppte anzeigen aktiv ist. Namen werden in Config gespeichert und aendern nur die Anzeige.

Luefterkurven

Oeffnen Sie Luefterkurven ueber Optionen > Luefterkurven... oder im Dialog Lueftersteuerung.

Jede Kurve waehlt eine beschreibbare Lueftersteuerung und einen Temperaturmesswert. Sensor Readout setzt den Luefter am niedrigen Temperaturpunkt auf den niedrigen Prozentwert, steigt zwischen niedrigem und hohem Punkt an und verwendet den Notfall-Prozentwert, sobald die Notfalltemperatur erreicht wird. Luefterkurven brauchen eine von LibreHardwareMonitor bereitgestellte beschreibbare Steuerung; wenn die Hardware nur RPM bereitstellt, kann Sensor Readout den Luefter lesen, aber nicht steuern.

Luefterprofile

Einstellungen > Luefterprofile erstellt benannte Gruppen von Luefteraktionen. Das Layout entspricht dem Hotkeys-Tab: Profil erstellen, optionalen globalen Hotkey setzen, Lueftersteuerungen hinzufuegen und pro Luefter manuelle Prozent oder Automatik/Standard waehlen.

Luefterprofile koennen auch als Umschalter markiert werden. Dann wendet der erste Druck auf den Profil-Hotkey das Profil an, und der zweite Druck setzt die Luefter dieses Profils wieder auf Automatik/Standard.

  • Alltag: Gehaeuseluefter leise setzen und den CPU-Luefter automatisch lassen.
  • Spiel oder Rendering: Gehaeuse-, CPU- und GPU-Luefter vor hoher Last staerker setzen.
  • Automatik wiederherstellen: ein Profil, das alle beschreibbaren Luefter wieder auf Automatik/Standard setzt.

Luefterprofile sind computerspezifisch, weil Lueftersteuerungen von der aktuellen Hardware abhaengen. Neue Installationen beginnen mit leeren Starterprofilen fuer Alltag, hohe Last und Automatik wiederherstellen; fuegen Sie passende Luefter hinzu, benennen Sie die Profile um oder loeschen Sie sie. Eine aktive Luefterkurve kann denselben Luefter bei der naechsten Aktualisierung wieder anpassen.

Beispiel-Setups

  • Schneller Status-Hotkey: CPU-Last, CPU-Temperatur, GPU-Temperatur und wichtige Netzwerkraten.
  • Laufwerksaktivitaet: Lese- und Schreibraten wichtiger Laufwerke, mit Doppeldruck zum Kopieren.
  • Leiser Desktop: CPU-Luefterkurve, leises Gehaeuseluefterprofil und Temperaturalarm.
  • USB-Fehlersuche: USB oeffnen, Enter auf einem Geraet druecken, Detailfelder auswaehlen und kopieren.

Berichte

Druecken Sie Strg+S oder Datei > Bericht speichern.... TXT gruppiert nach Typ und Geraet. HTML verwendet Tabellen mit Sensor, Wert und Quelle.

Dateien

Portable Build:

  • Sensor Readout.exe: Hauptprogramm.
  • Install-Prerequisites.cmd und Install-Prerequisites.ps1: optionale Helfer fuer .NET Framework, PawnIO und Fehlersuche.

Von Sensor Readout verwendete Ordner:

  • Config\Shared.json: portable Einstellungen, die zwischen Computern geteilt werden, z. B. Sprache, Aktualisierung, Temperatureinheit, Start-/Beenden-Klaenge, Startansage, Infobereich-Anzeige, Updatepruefung und globale Hotkeys.
  • Config\<ComputerName>.json: computerspezifische Einstellungen wie Windows-Start, ausgewaehlte Infobereich-Messwerte, Messwertlisten fuer gesprochene Hotkeys, Alarme, ausgeblendete Messwerte, Luefternamen, Lueftersteuerung, eigene gesprochene Messwertnamen, Protokollierungsstufe und Erststart-Hinweise.
  • Logs\<ComputerName>.log: optionale Diagnose, Luefteraktionen, Hotkey-Registrierung und Screenreader-Sprachnachrichten.
  • Reports: Standardordner fuer Berichte aus der Befehlszeile oder dem ersten Speichern-Dialog.
  • Langs: bearbeitbare Sprachdateien.
  • Docs: mitgelieferte HTML-Handbuecher.
  • Sounds: mitgelieferte WAV-Dateien wie SR01.wav und eigene WAV-Dateien.
  • Plug-Ins: optionale Hardware-Plug-Ins. Mitgelieferte Plug-Ins wie Framework Laptop und experimentelle HP-Hardware-Unterstuetzung bleiben deaktiviert, bis sie in den Einstellungen aktiviert werden.
  • Data\usb.ids: USB-Hersteller-/Produktdaten.
  • Data\oui.csv: MAC/OUI-Herstellerdaten fuer Netzwerkadapter.

Sprachdateien

  • Legen Sie .txt- oder .ini-Dateien in Langs.
  • Beim ersten Start waehlt Sensor Readout eine Sprache passend zur Windows-Anzeigesprache oder Englisch.
  • Verwenden Sie key=value; # und ; sind Kommentare.
  • language.name ist der Anzeigename.
  • manual.file=README-xx.html legt das F1-Handbuch fest. Nur Dateiname, kein Pfad. Gesucht wird in Docs, sonst faellt Sensor Readout auf Englisch zurueck.
  • number.decimalSeparator=, setzt den Sprachstandard fuer Dezimalwerte.
  • Der Ordner wird etwa alle 15 Sekunden geprueft.
  • Langs\English.txt ist die primaere Standardsprache und Vorlage fuer neue Sprachen.
  • Das Dezimaltrennzeichen kann auch in den Einstellungen geaendert werden.

Optionale Screenreader-Sprache verwendet Tolk.dll, SAAPI64.dll und Tolk-Unterstuetzungs-DLLs, wenn Screenreader laeuft. Beim minimierten Start kann eine konfigurierbare Startansage gesprochen werden. Der Spracheditor erlaubt Bearbeitung ohne externen Texteditor.

Hinweise

Luefterunterstuetzung haengt von LibreHardwareMonitor, PawnIO, Sensorchip und BIOS ab. Manche Systeme liefern Messwerte, aber keine schreibbaren Lueftersteuerungen. Fuer Mainboard-Super-I/O-Zugriff sind oft Administratorrechte noetig.

Wenn CPU-Temperatur oder CPU-Last fehlen, kann das Installieren und Starten von Core Temp helfen. Sensor Readout kann die Shared-Memory-Daten von Core Temp lesen, wenn Core Temp bereits laeuft. Core Temp ist optional und wird nur als Ersatzquelle fuer CPU-Messwerte verwendet. Nutzen Sie Hilfe > Core-Temp-Unterstuetzung installieren..., um die offizielle Core-Temp-Seite zu oeffnen oder die Installation mit winget oder Chocolatey zu starten, wenn diese Werkzeuge verfuegbar sind.

Wenn Lueftersteuerungen fehlen, oeffnen Sie Optionen > Lueftersteuerung... und aktivieren Sie Gestoppte anzeigen. Manche Mainboards melden steuerbare Anschluesse als gestoppt oder undefiniert, bis sie drehen; diese Option macht sie zum Testen sichtbar.

Framework-Laptops

Framework-Laptop-Besitzer, die Framework-spezifische Luefter-RPM und Temperaturen sehen moechten, sollten zuerst das BIOS ueber Framework BIOS and Drivers Downloads aktualisieren und dann Framework Control installieren und starten. Wenn Framework Control im Browser Luefter- und Temperatursteuerungen zeigt, oeffnen Sie Optionen > Einstellungen > Plug-Ins, aktivieren Sie das Framework-Laptop-Plug-In und schliessen Sie Einstellungen. Danach kann Sensor Readout die lokale API lesen und Framework Control-Temperatur- und Luefterzeilen anzeigen. Sensor Readout liest diese lokale API nur; es installiert Framework Control nicht, aendert darueber keine Lueftereinstellungen und flasht keine Firmware.
