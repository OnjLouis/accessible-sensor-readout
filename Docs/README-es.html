Sensor Readout

Version actual: 3.7.0.

Sensor Readout es una utilidad de Windows para leer sensores de hardware y controlar ventiladores compatibles con una interfaz pensada primero para teclado y lectores de pantalla.

A la izquierda muestra categoras principales, a la derecha una vista de rbol con lecturas agrupadas por dispositivo o finalidad, y los comandos comunes estn en una barra de mens normal.

Sensor Readout se puede encontrar en GitHub.

Por que existe Sensor Readout

Toda esa informacion sobre tu propio ordenador que antes era dificil de encontrar, escondida detras del Administrador de dispositivos o encerrada en apps caras o inaccesibles, deberia ser mas facil de alcanzar.

Sensor Readout busca reunir tanta de esa informacion como sea posible en una app gratuita, facil de usar, portable o instalable, para que no tengas que buscarla por todas partes. Estaba harto del estado de la accesibilidad en la parte tecnica de la informatica con Windows, asi que quise crear algo que yo mismo quisiera usar. Al parecer, muchas otras personas tambien querian eso.

Espero que descubras tanto sobre tu ordenador con esta app como yo he descubierto sobre el mio, y espero que te sorprenda gratamente lo que encuentres.

Mi deseo es que Sensor Readout se convierta en el referente de su clase: una herramienta pensada primero para la accesibilidad, que las personas ciegas y con baja vision que trabajan en soporte tecnico puedan entregar a sus clientes para que generen informes y reciban ayuda para resolver problemas. Busco la excelencia, y espero que lo notes al usarla.

Qu hace

  • Lee temperaturas, RPM de ventiladores, salud y capacidad de almacenamiento, y contadores de hardware seleccionados.
  • Muestra una categoria Dispositivos con inventario PnP de Windows, incluidos dispositivos PCI/sistema, controladores de almacenamiento, dispositivos de entrada, Bluetooth, impresoras, dispositivos de seguridad y entradas breves relacionadas para hardware USB, audio, pantalla y red.
  • Coloca los dispositivos con codigos de problema de Windows, fallos de controlador o estado no OK en un grupo claro Dispositivos que no funcionan al principio de Dispositivos, manteniendolos tambien en su grupo normal de hardware.
  • Muestra una pgina Rendimiento/Vista general con tiempo activo, BIOS, GPU, uso de CPU, uso de memoria y actividad de lectura/escritura de unidades.
  • Abre la ventana principal de inmediato mientras la primera actualizacin de sensores contina en segundo plano.
  • Muestra una categora Red con estado del adaptador, direcciones IP, velocidad de enlace, tasas de envo/recepcin y trfico total.
  • Muestra direcciones MAC y fabricantes OUI de adaptadores de red cuando la base OUI incluida contiene el prefijo.
  • Muestra una categora USB para dispositivos conectados, hubs, controladores, velocidad de conexin, consumo donde Windows lo expone, letras de unidad, estado de extraccin segura y campos de detalle copiables.
  • Muestra carga de batera del porttil, estado, capacidad, salud, ciclos, voltaje y tasa de energa cuando Windows los proporciona.
  • Admite curvas simples de ventilador que ajustan un control desde una lectura de temperatura seleccionada.
  • Admite perfiles de ventilador que aplican varios controles a la vez, con atajos de teclado globales opcionales y opcin de volver a automtico al pulsar otra vez.
  • Enlaza la pgina del proyecto desde README, men Ayuda y dilogo Acerca de.
  • Usa bibliotecas LibreHardwareMonitor incluidas para acceder a sensores.
  • Usa el controlador PawnIO para sensores de placa base y control de ventiladores de bajo nivel cuando el hardware lo permite.
  • Permite poner nombres fciles a cabeceras de ventilador.
  • Oculta por defecto cabeceras de ventilador detenidas o sin usar.
  • Aplica porcentajes manuales a un ventilador seleccionado o a todos los ventiladores visibles.
  • Devuelve un ventilador o todos a control automtico/predeterminado.
  • Guarda informes TXT o HTML.
  • Abre informes guardados directamente, incluidos informes TXT o HTML enviados dentro de un archivo ZIP.
  • Permite configurar la actualizacin automtica y usa 5 segundos como valor predeterminado en configuraciones nuevas.
  • Puede ejecutarse al iniciar Windows e iniciar minimizado en el rea de notificacin.
  • Usa Config\Shared.json para preferencias portables que deben seguir a la app entre equipos, y mantiene la configuracin especfica del hardware en Config\<ComputerName>.json.
  • Guarda los cambios de preferencias al momento para proteger trabajo de atajos de teclado, bandeja, elementos ocultos y opciones similares.
  • Escribe diagnstico en Logs\<ComputerName>.log cuando el registro est activado.
  • Puede mostrar temperaturas en Celsius, Fahrenheit, Celsius luego Fahrenheit o Fahrenheit luego Celsius.
  • Admite atajos de teclado globales opcionales para mostrar/ocultar y leer el estado del rea de notificacin.
  • Puede hablar el estado del rea de notificacin mediante lector de pantalla con las DLL de biblioteca de lector de pantalla Tolk de 64 bits incluidas.
  • Admite archivos de idioma editables en la carpeta Langs.
  • El registro est desactivado por defecto y se puede activar para solucionar problemas.
  • Puede mostrar lecturas seleccionadas en el tooltip del rea de notificacin.

Requisitos

  • Windows 10 o Windows 11, 64 bits.
  • Microsoft .NET Framework 4.8 o posterior.
  • Controlador PawnIO instalado para sensores de placa base y control de ventiladores.
  • Derechos de administrador para leer muchos sensores de placa base o controlar ventiladores.

No hace falta ejecutar LibreHardwareMonitor como aplicacion separada porque esta carpeta incluye las bibliotecas necesarias. Instalar la aplicacion independiente puede ser til para diagnstico.

Instalador de requisitos

Ejecute primero Install-Prerequisites.cmd desde esta carpeta. Llama a Install-Prerequisites.ps1, pide permisos de administrador si hace falta e instala PawnIO. Prueba winget primero, despues Chocolatey si ya esta instalado, y si no descarga la version oficial de PawnIO.Setup desde GitHub. Cuando winget existe, tambien puede instalar .NET Framework Runtime.

Install-Prerequisites.cmd

Para instalar tambien la app independiente LibreHardwareMonitor con winget para diagnostico, ejecute:

.\Install-Prerequisites.ps1 -IncludeLibreHardwareMonitor

Comandos de instalacion manual

Si el instalador de requisitos no funciona en su sistema, abra Windows Terminal o PowerShell como administrador y ejecute estos comandos:

winget install --id Microsoft.DotNet.Framework.Runtime -e --source winget

winget install --id namazso.PawnIO -e --source winget

Herramienta opcional de diagnostico:

winget install --id LibreHardwareMonitor.LibreHardwareMonitor -e --source winget

Primeros pasos

1. Inicie Sensor Readout.exe.

2. Acepte el aviso de administrador de Windows.

3. Si falta PawnIO, Sensor Readout ofrece ejecutar el instalador de requisitos.

4. Pulse F5 si los valores an se estn cargando.

5. Abra Opciones > Controles de ventilador... o pulse Ctrl+L para ajustar ventiladores.

Si faltan ventiladores o controles de placa base, compruebe PawnIO y que la app se haya iniciado como administrador. El instalador se puede volver a ejecutar desde Ayuda > Instalar requisitos....

Preferencias

Use Opciones > Preferencias o Ctrl+,. El cuadro recuerda la pestana usada por ultima vez en la sesion actual.

General (Ctrl+1)

Idioma, actualizacion, intervalo, icono del area de notificacion, unidad de temperatura, separador decimal, nivel de registro, busqueda de actualizaciones y lecturas del area de notificacion.

Inicio e instalacion (Ctrl+2)

Instalacion en este PC, inicio con Windows, inicio minimizado, mensaje hablado de inicio, sonidos de inicio/cierre y feedback de diagnostico.

Instalar en este PC... copia la carpeta portable actual a la carpeta de programas de Windows de este usuario, puede crear un acceso directo en el escritorio, cierra la copia antigua e inicia la copia instalada. Si el inicio con Windows esta activado, el acceso directo de inicio apunta despues a la copia instalada.

Cuando Sensor Readout ya se ejecuta desde la ubicacion instalada, este boton cambia a Desinstalar de este PC.... La desinstalacion quita los archivos instalados y accesos directos, pero deja Config, Logs y Reports. La misma pestana tambien incluye una casilla de acceso directo del escritorio, para agregarlo o quitarlo sin reinstalar.

Si Sensor Readout se reinicia despues de instalar o actualizar y no aparece la ventana principal, compruebe si esta activado Iniciar minimizado al area de notificacion. Windows tambien puede ocultar el icono hasta que lo permita en la configuracion del area de notificacion o bandeja del sistema. Al mostrar el icono de Sensor Readout, podra abrir la ventana principal desde ese icono.

Si el estado del area de notificacion esta activado, minimizar oculta Sensor Readout de la barra de tareas y Alt+Tab. Abralo desde el icono. Alt+F4 sale de la app.

Las lecturas del area de notificacion se mueven de una lista disponible a una lista de orden. Windows muestra tooltips cortos, por eso el limite es cuatro. Una lectura aparece en una sola lista. Los perfiles importados no conservan sus teclas antiguas, para no robar teclas ya usadas en este equipo. Las lecturas solo se conservan cuando Sensor Readout puede emparejarlas con seguridad en el equipo actual.

Leer sensores

El panel de lecturas es una vista de rbol. Temperaturas, Ventiladores, SMART, Rendimiento y Red agrupan por dispositivo o finalidad y despus por valor.

La lista izquierda solo cambia la vista; no activa ni desactiva dispositivos. Sensor Readout inicia en Rendimiento/Vista general. Rendimiento agrupa CPU, memoria y actividad de unidades. Red muestra estado, IP, velocidad, tasas y contadores.

Las lecturas de porcentaje tambin se exponen en la barra de progreso inferior para que los lectores de pantalla usen su feedback habitual. Use Edicion, tecla Aplicaciones o clic derecho para copiar, abrir Detalles, renombrar un ventilador u ocultar valores.

Alarmas y sonidos

Preferencias > Alarmas permite crear alarmas de lecturas. Elija una lectura, Mayor o igual, Menor o igual, o Igual, y despus el umbral y la espera. Cada alarma puede hablar con lector de pantalla, reproducir un WAV, o ambas cosas.

Preferencias > Inicio e instalacion incluye sonidos opcionales de inicio y salida. Sensor Readout carga WAV desde la carpeta Sounds. Los sonidos incluidos usan nombres neutros como SR01.wav, SR02.wav, etc.; los sonidos agregados por el usuario pueden tener cualquier nombre normal de archivo .wav. Cualquier sonido de la carpeta puede elegirse en cualquier lugar donde Sensor Readout ofrezca un sonido, incluidos alarmas, inicio o cierre, diagnosticos, avisos de actualizacion y acciones de perfiles de ventilador.

Plug-Ins

Preferencias > Plug-Ins muestra los plug-ins de hardware instalados y explica qu hace cada uno. Los plug-ins son cdigo de confianza; active solo plug-ins de personas o proyectos en los que confe.

Los plug-ins instalados empiezan desactivados. Importar un ZIP copia el plug-in a la carpeta Plug-Ins, pero no lo activa automticamente.

Cuando un plug-in activado incluye paginas de soporte relacionadas, esos enlaces aparecen en el menu Ayuda. Por ejemplo, los plug-ins de portatil incluidos pueden mostrar las paginas de Dell Command | Monitor, G-Helper, HP Support Assistant, OMEN Gaming Hub, Framework BIOS and Drivers o Framework Control.

El plug-in incluido para porttiles Framework es opcional y est desactivado por defecto. Puede anadir temperaturas y RPM de ventilador de Framework cuando Framework Control est instalado y en ejecucin.

Para desarrolladores, el arbol de codigo fuente en GitHub incluye Docs\\Plug-In-development.md.

Use Editar > Buscar lectura... o F3 para buscar lecturas en todas las categorias. La busqueda se reduce al escribir. Tab va a la lista de resultados, Intro se mueve a la lectura seleccionada, Alt+L limpia la busqueda, y Esc o Cerrar vuelve a la ventana principal.

Flujo de control de ventiladores

Abra Opciones > Controles de ventilador... o pulse Ctrl+L.

1. Elija un control. 2. Opcional: escriba un nombre y pulse Alt+L. 3. Escriba un porcentaje de 0 a 100. 4. Pulse Alt+M. 5. Alt+A vuelve el seleccionado a automtico. 6. Alt+R vuelve todos los visibles a automtico.

Los botones para todos los ventiladores solo afectan controles visibles. Las cabeceras detenidas o vacias estn ocultas hasta activar Mostrar detenidos. Los nombres se guardan en la configuracin especfica del equipo dentro de Config y solo cambian como aparecen en Sensor Readout.

Curvas de ventilador

Abra las curvas desde Opciones > Curvas de ventilador... o desde el dilogo Control de ventiladores.

Cada curva elige un control de ventilador escribible y una lectura de temperatura. Sensor Readout usa el porcentaje bajo en la temperatura baja, sube progresivamente entre los puntos bajo y alto, y usa el porcentaje de emergencia cuando se alcanza la temperatura de emergencia. Las curvas dependen de que LibreHardwareMonitor exponga un control escribible; si el hardware solo expone RPM, Sensor Readout puede leer el ventilador pero no controlarlo.

Perfiles de ventilador

Preferencias > Perfiles de ventilador permite crear grupos nombrados de acciones de ventilador. El diseno coincide con la pestaa Atajos de teclado: crear perfil, asignar atajo global opcional, agregar controles y elegir porcentaje manual o automtico/predeterminado para cada ventilador.

Los perfiles de ventilador tambin pueden marcarse como conmutadores. Con esa opcin, una pulsacion aplica el perfil y la siguiente vuelve esos ventiladores a automtico/predeterminado.

  • Uso diario: ventiladores de caja silenciosos y CPU en automtico.
  • Juegos o renderizado: subir caja, CPU y GPU antes de una carga fuerte.
  • Volver a automtico: un perfil que devuelve todos los controles escribibles a automtico/predeterminado.

Los perfiles son especificos de cada equipo porque las claves de control dependen del hardware actual. Las instalaciones nuevas empiezan con perfiles iniciales vacios para uso diario, cargas fuertes y volver a automtico; agregue los ventiladores adecuados, cambie nombres o elimine esos perfiles. Una curva activa puede volver a ajustar el mismo ventilador en la siguiente actualizacin.

Ejemplos de configuracin

  • Atajo de estado rpido: uso CPU, temperatura CPU, temperatura GPU y tasas de red importantes.
  • Actividad de disco: lectura y escritura de las unidades importantes, con doble pulsacion para copiar.
  • Escritorio silencioso: curva de CPU, perfil silencioso de caja y alarma de temperatura.
  • Diagnostico USB: abrir USB, Enter en un dispositivo, seleccinar campos tiles y copiarlos.

Diagnsticos

Use Ayuda > Ejecutar diagnosticos... o pulse Alt+F1 cuando necesite reunir informacion de soporte. Sensor Readout crea un archivo ZIP en Reports con el nombre del equipo en el nombre del archivo, abre la carpeta al terminar y elimina la carpeta temporal despues de crear el ZIP.

El ZIP de diagnostico contiene un informe TXT, un informe HTML, un registro de depuracion, un resumen de diagnostico e informacion de tiempos. La prueba comprueba brevemente los controles de ventilador escribibles al 100% y luego restaura cada ventilador a su estado anterior manual, automatico/predeterminado o de curva de ventilador.

Los diagnosticos pueden decir el progreso y reproducir sonidos de inicio/final. Configure esto en Opciones > Preferencias > Inicio. Los diagnosticos por linea de comandos usan las mismas preferencias salvo que use --diagnostics-quiet, --no-diagnostics-speech o --no-diagnostics-sounds.

Para pruebas desatendidas, ejecute Sensor Readout.exe --diagnostics [ruta]. Si [ruta] es una carpeta, el ZIP se crea alli. Si no se indica ruta, Sensor Readout crea el ZIP en Reports.

Informes

Los informes sirven para dos situaciones: guardar una instantanea de su propio equipo o enviar esa instantanea a otra persona para que pueda verla dentro de Sensor Readout.

Guardar un informe

Pulse Ctrl+S o use Archivo > Guardar informe....

El dialogo de guardado ofrece informes TXT y HTML. TXT agrupa por tipo de lectura y dispositivo, para que los nombres largos de hardware se escriban una vez como encabezados. HTML usa tablas con sensor, valor y fuente.

El primer guardado crea la carpeta Reports si no existe. Para compartir un informe, envie el archivo .html o .txt guardado. HTML suele ser mas comodo de leer en un navegador; TXT es util para pegar en mensajes. Si alguien necesita un paquete de soporte mas completo, use Ayuda > Ejecutar diagnosticos...; los diagnosticos incluyen ambos formatos, registros y un resumen en un ZIP.

Abrir el informe de otro equipo

Pulse Ctrl+O o use Archivo > Abrir informe... para cargar un informe TXT o HTML guardado por Sensor Readout. Si alguien envia un informe dentro de un archivo ZIP, abra el ZIP directamente y Sensor Readout usara el primer informe legible que encuentre dentro.

El informe se abre como una instantanea estatica en la vista normal de categorias y arbol. Asi puede inspeccionar el equipo de otra persona como si fuera su vista actual. El modo de informe estatico no actualiza valores en vivo, no ejecuta alarmas y no controla hardware. Solo muestra los datos guardados en el informe.

Los detalles y comandos de copia siguen funcionando cuando el informe contiene los campos necesarios. Esto ayuda cuando alguien envia datos USB, audio, pantalla, red, ventiladores, bateria o SMART y usted quiere revisar filas concretas sin leer todo el archivo manualmente.

Use Archivo > Volver a lecturas en vivo o Ctrl+R para volver al equipo actual. Los informes HTML guardados por Sensor Readout 2.0 o posterior incluyen una instantanea oculta legible por la app para una mejor importacion. Los informes TXT permanecen como texto normal y se abren lo mejor posible.

Transferencia de configuracion

Use Archivo > Exportar configuracion y perfiles... o Ctrl+E para crear un paquete de configuracion. Sensor Readout muestra primero una lista de comprobacion, para que pueda elegir exactamente que incluir. Use Archivo > Importar configuracion y perfiles... o Ctrl+Mayus+I para recuperar configuracion. La importacion muestra el mismo tipo de lista, limitada a las secciones encontradas en el paquete seleccionado. Las lecturas portables como memoria, CPU, tiempo activo del sistema y muchos valores de resumen se emparejan cuando es posible. Los elementos especificos del hardware se importan con cuidado: las teclas globales quedan vacias, las acciones de perfiles de ventilador no se vinculan, las curvas de ventilador quedan desactivadas hasta elegir un control de ventilador, y las alarmas quedan desactivadas para revision.

Use Archivo > Exportar copia portable... o Ctrl+Mayus+E cuando quiera guardar toda la app personalizada como ZIP. Incluye Sensor Readout, Config, plug-ins, archivos de idioma, sonidos personalizados y los demas archivos necesarios para ejecutarla desde otra carpeta u otro PC. Reports y Logs son casillas opcionales porque pueden ser grandes o privados.

Archivos

Build portable:

  • Sensor Readout.exe: aplicacion principal.
  • Install-Prerequisites.cmd y Install-Prerequisites.ps1: ayudantes opcionales para .NET Framework, PawnIO y diagnstico.

Carpetas usadas por Sensor Readout:

  • Config\Shared.json: preferencias portables compartidas entre equipos, como idioma, actualizacin, unidad de temperatura, sonidos, voz de inicio, visibilidad del rea de notificacin, actualizaciones y atajos de teclado globales.
  • Config\<ComputerName>.json: preferencias especficas del equipo, como inicio con Windows, lecturas de rea de notificacin, atajos de teclado hablados, alarmas, lecturas ocultas, nombres de ventilador, control de ventiladores, nombres hablados personalizados, logging y avisos de primer inicio.
  • Logs\<ComputerName>.log: diagnstico opcional, acciones de ventilador, atajos de teclado y mensajes de screen reader.
  • Reports: carpeta predeterminada para informes de lnea de comandos o del primer dilogo Guardar.
  • Langs: archivos de idioma editables.
  • Docs: manuales HTML incluidos.
  • Sounds: sonidos WAV incluidos como SR01.wav y WAV propios.
  • Plug-Ins: plug-ins opcionales de hardware. Los incluidos, como Framework Laptop y los experimentales HP Hardware Support, Dell Latitude Support y Asus ROG/TUF Support, estn desactivados hasta activarlos en Preferencias. El plug-in Asus incluye su propio aviso GPL.
  • Data\usb.ids: datos USB de fabricantes/productos.
  • Data\oui.csv: datos MAC/OUI para adaptadores de red.

Archivos de idioma

  • Ponga archivos .txt o .ini en Langs.
  • En el primer inicio se elige el idioma de Windows si existe, o ingles.
  • Use lineas key=value; # y ; son comentarios.
  • language.name es el nombre mostrado.
  • manual.file=README-xx.html indica el manual para F1. Solo nombre de archivo, sin ruta. Se busca en Docs y si falla se usa ingles.
  • number.decimalSeparator=, define el separador decimal predeterminado del idioma.
  • La carpeta se revisa cada 15 segundos aproximadamente.
  • Langs\English.txt es el idioma principal y plantilla para idioms nuevos.
  • El separador decimal tambin se puede cambiar en Preferencias.

La voz opcional de lector de pantalla usa Tolk.dll, SAAPI64.dll y DLL de soporte de Tolk si lector de pantalla esta en ejecucin. Al iniciar minimizado puede hablar un mensaje configurable. El editor de idioma permite editar sin abrir otro editor.

Notas

La compatibilidad de ventiladores depende de LibreHardwareMonitor, PawnIO, el chip sensor y la BIOS. Algunos sistemas dan lecturas pero no controles escribibles. Para Super I/O de placa base normalmente se requieren derechos de administrador.

Si faltan la temperatura o la carga de CPU, puede ayudar instalar y ejecutar Core Temp. Sensor Readout puede leer los datos de memoria compartida de Core Temp cuando Core Temp ya esta en ejecucin. Core Temp es opcional y solo se usa como fuente alternativa para lecturas de CPU. Use Ayuda > Instalar soporte Core Temp... para abrir la pgina oficial de Core Temp o iniciar la instalacion con winget o Chocolatey cuando esas herramientas esten disponibles.

Si no aparecen controles de ventilador, abra Opciones > Controles de ventilador... y active Mostrar detenidos. Algunas placas informan cabeceras controlables como detenidas o indefinidas hasta que empiezan a girar; esta opcin las muestra para probar.

Plug-ins de marca de portatil

Los usuarios de portatiles Framework, Dell, Lenovo, Asus, HP, OMEN y Victus pueden obtener mejor visibilidad de temperaturas o ventiladores activando el plug-in de hardware correspondiente en Opciones > Preferencias > Plug-Ins. Los plug-ins estan desactivados hasta que los active, y los cambios se aplican al cerrar Preferencias.

Los usuarios de Framework que quieran RPM de ventilador y temperaturas especificas del modelo deben actualizar primero la BIOS desde Framework BIOS and Drivers Downloads, instalar y ejecutar Framework Control. Si Framework Control muestra controles de ventilador y temperatura en su interfaz de navegador, active el plug-in Framework Laptop en Sensor Readout.

Los plug-ins Dell, Lenovo, Asus, HP, OMEN y Victus no garantizan lecturas de ventilador en todos los modelos, pero conviene probarlos antes de enviar diagnosticos o asumir que el portatil no expone mas datos de sensores.

Las herramientas opcionales del fabricante tambien pueden ayudar a exponer o comprobar datos especificos del portatil. Los usuarios de Dell pueden probar Dell Command | Monitor. Los usuarios de Lenovo pueden probar Lenovo Vantage o Lenovo System Update. Los usuarios de Asus pueden probar G-Helper. Los usuarios de HP, OMEN y Victus pueden probar HP Support Assistant y OMEN Gaming Hub. Estas herramientas no forman parte de Sensor Readout; use las paginas del fabricante o del proyecto e instale solo lo que tenga sentido para su equipo.

Sensor Readout solo lee estas rutas de soporte opcionales salvo que un plug-in indique otra cosa; no actualiza firmware ni reemplaza las herramientas del fabricante.

Report Comparison, Support, And History

Sensor Readout 3.6.0 adds File > Compare reports... (Ctrl+Shift+M) for before-and-after or two-machine comparisons, File > Save anonymized report... (Ctrl+Shift+A) for masking common private identifiers before sharing, and Help > Prepare support report... for creating a diagnostic ZIP with a copyable path and GitHub issue shortcut.

Use Options > Enable reading history logging, then select a reading and press Ctrl+Shift+G or choose Edit > Add to history log to write selected readings to a short-term CSV log in Logs. Performance/Overview also includes a Data sources group showing which Windows, LibreHardwareMonitor, plug-in, database, and Sensor Readout sources contributed to the current view.
