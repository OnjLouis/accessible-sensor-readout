Sensor Readout

Versione attuale: 2.0.0.

Sensor Readout e un'utilita Windows per leggere sensori hardware e controllare ventole compatibili, con interfaccia pensata prima di tutto per tastiera e screen reader.

A sinistra mostra categorie principali, a destra una vista ad albero con letture raggruppate per dispositivo o scopo, e i comandi comuni sono in una normale barra menu.

Sensor Readout e disponibile su GitHub.

Cosa fa

  • Legge temperature, RPM ventole, salute e capacita storage, e contatori hardware selezionati.
  • Mostra Prestazioni/Panoramica con uptime, BIOS, GPU, uso CPU, uso memoria e attivita lettura/scrittura delle unita.
  • Apre subito l'interfaccia principale mentre il primo aggiornamento sensori continua in background.
  • Mostra rete con stato adattatore, indirizzi IP, velocita collegamento, invio/ricezione e traffico totale.
  • Mostra indirizzi MAC e produttori OUI degli adattatori di rete quando il database OUI incluso contiene il prefisso.
  • Mostra una categoria USB per dispositivi collegati, hub, controller, velocita di connessione, corrente dove Windows la espone, lettere unita, stato rimozione sicura e campi dettagliati copiabili.
  • Mostra carica batteria, stato, capacita, salute, cicli, tensione e potenza quando Windows li fornisce.
  • Supporta curve ventole semplici che impostano un controllo ventola da una lettura temperatura selezionata.
  • Supporta profili ventole che applicano piu controlli insieme, con hotkey globali opzionali e ritorno automatico opzionale alla pressione successiva.
  • Collega la pagina progetto da README, menu Aiuto e finestra Informazioni.
  • Usa librerie LibreHardwareMonitor incluse per l'accesso ai sensori.
  • Usa il driver PawnIO per sensori scheda madre e controllo ventole dove l'hardware lo supporta.
  • Permette nomi facili per intestazioni ventola.
  • Nasconde per impostazione predefinita intestazioni ferme o non popolate.
  • Applica percentuali manuali a una ventola selezionata o a tutte le ventole visibili.
  • Riporta una o tutte le ventole al controllo automatico/predefinito.
  • Salva report TXT o HTML.
  • Supporta aggiornamento automatico configurabile e usa 5 secondi come intervallo iniziale.
  • Puo avviarsi con Windows e partire minimizzato nell'area di notifica.
  • Usa Config\Shared.json per preferenze portabili che devono seguire l app tra computer, mantenendo la configurazione specifica dell hardware in Config\<ComputerName>.json.
  • Salva subito le modifiche alle preferenze, proteggendo hotkey, area di notifica, elementi nascosti e opzioni simili.
  • Scrive diagnostica in Logs\<ComputerName>.log quando il logging e attivo.
  • Mostra temperature in Celsius, Fahrenheit, Celsius poi Fahrenheit o Fahrenheit poi Celsius.
  • Supporta hotkey globali opzionali per mostrare/nascondere e leggere lo stato dell'area di notifica.
  • Puo parlare tramite screen reader usando le DLL libreria screen reader Tolk a 64 bit incluse.
  • Supporta file lingua modificabili nella cartella Langs.
  • Il logging e disattivato per impostazione predefinita e si puo abilitare per diagnosi.
  • Puo mostrare letture scelte nel tooltip dell'area di notifica.

Requisiti

  • Windows 10 o Windows 11, 64 bit.
  • Microsoft .NET Framework 4.8 o successivo.
  • Driver PawnIO installato per sensori scheda madre e controllo ventole.
  • Diritti amministratore per leggere molti sensori scheda madre o controllare ventole.

LibreHardwareMonitor non deve essere eseguito come app separata perche questa cartella include le librerie necessarie. L'app autonoma puo essere utile per diagnosi.

Installatore requisiti

Eseguire prima Install-Prerequisites.cmd da questa cartella. Richiama Install-Prerequisites.ps1, chiede privilegi amministratore se necessario e installa PawnIO. Prova prima winget, poi Chocolatey se e gia installato, quindi scarica la versione ufficiale PawnIO.Setup da GitHub se nessun gestore pacchetti e disponibile. Quando winget esiste, puo installare anche .NET Framework Runtime.

Install-Prerequisites.cmd

Per installare anche l app standalone LibreHardwareMonitor con winget per la diagnosi, eseguire:

.\Install-Prerequisites.ps1 -IncludeLibreHardwareMonitor

Comandi manuali di installazione

Se l installatore requisiti non funziona sul sistema, aprire Windows Terminal o PowerShell come amministratore ed eseguire questi comandi:

winget install --id Microsoft.DotNet.Framework.Runtime -e --source winget

winget install --id namazso.PawnIO -e --source winget

Strumento opzionale di diagnosi:

winget install --id LibreHardwareMonitor.LibreHardwareMonitor -e --source winget

Primi passi

1. Avviare Sensor Readout.exe.

2. Accettare la richiesta amministratore di Windows.

3. Se PawnIO manca, Sensor Readout propone l'installatore dei requisiti.

4. Premere F5 se i valori stanno ancora caricando.

5. Aprire Opzioni > Controllo ventole... o premere Ctrl+L.

Se mancano ventole o controlli scheda madre, controllare PawnIO e l'avvio come amministratore. L'installatore e disponibile piu tardi da Aiuto > Installa requisiti....

Preferenze

Usare Opzioni > Preferenze o Ctrl+,. La finestra ricorda l ultima scheda usata nella sessione.

Generale (Ctrl+1)

Lingua, aggiornamento, intervallo, icona area di notifica, unita temperatura, separatore decimale, livello logging, controllo aggiornamenti e letture area di notifica.

Avvio (Ctrl+2)

Avvio con Windows, avvio minimizzato, messaggio vocale di avvio, suoni di avvio/chiusura e feedback diagnostico.

Hotkey (Ctrl+3)

Mostra/nascondi globale, lettura area di notifica, profili hotkey parlati, copia con doppia pressione e nomi parlati brevi.

Profili ventola (Ctrl+4)

Gruppi di azioni ventola con hotkey opzionale, modalita toggle, messaggio parlato e suono.

Allarmi (Ctrl+5)

Monitorare letture, soglia, unita, cooldown, voce e suono.

Plug-Ins (Ctrl+6)

Vedere, descrivere, importare e attivare plug-in hardware installati.

Elementi nascosti (Ctrl+7)

Ripristinare letture o gruppi nascosti dalla finestra principale.

Editor lingua (Ctrl+8)

Modificare file lingua o crearne uno da inglese. Le stringhe mancanti ricadono sull inglese.

Quando lo stato area di notifica e attivo, minimizzare rimuove Sensor Readout da barra applicazioni e Alt+Tab. Riaprirlo dall icona. Alt+F4 chiude l app.

Le letture area di notifica passano da una lista disponibile a un ordine. Il limite e quattro perche i tooltip Windows sono brevi. Una lettura appare in una sola lista. Usare pulsanti o Ctrl+frecce. Gli elementi mostrano dispositivo, nome e categoria, ad esempio Ethernet - Rx: Rete.

La scheda Hotkey puo importare profili di hotkey parlate da un altra configurazione computer. I profili importati non conservano le vecchie combinazioni, cosi non rubano tasti gia usati su questo computer. Le letture vengono conservate solo quando Sensor Readout puo abbinarle in modo sicuro sul computer attuale.

Lettura sensori

Il pannello letture e una vista ad albero. Temperature, Ventole, SMART, Prestazioni e Rete raggruppano per dispositivo o scopo e poi per valore.

La lista sinistra cambia solo vista; non abilita ne disabilita dispositivi. Sensor Readout parte su Prestazioni/Panoramica. Prestazioni raggruppa CPU, memoria e attivita disco. Rete mostra stato, IP, velocita, tassi e contatori.

Le percentuali sono esposte anche nella barra di avanzamento inferiore per gli screen reader. Usare Modifica, tasto Applicazioni o clic destro per copiare, rinominare una ventola o nascondere valori.

Allarmi e suoni

Preferenze > Allarmi permette di creare allarmi per letture. Scegliere una lettura, Maggiore o uguale, Minore o uguale, o Uguale, poi soglia e attesa. Ogni allarme puo parlare con screen reader, riprodurre un WAV, o entrambe le cose.

Preferenze > Avvio include suoni opzionali di avvio e chiusura. Sensor Readout carica WAV dalla cartella Sounds. I suoni inclusi usano nomi neutri come SR01.wav, SR02.wav e cosi via; i suoni aggiunti dall'utente possono avere qualsiasi normale nome file .wav. Qualsiasi suono nella cartella puo essere scelto ovunque Sensor Readout offra un suono, inclusi allarmi, avvio o chiusura, diagnostica, avvisi di aggiornamento e azioni dei profili ventola.

Plug-Ins

Preferenze > Plug-Ins elenca i plug-in hardware installati e descrive cosa fanno. I plug-in sono codice considerato attendibile; abilita solo plug-in da persone o progetti fidati.

I plug-in installati partono disattivati. Importare uno ZIP copia il plug-in nella cartella Plug-Ins, ma non lo abilita automaticamente.

Il plug-in Framework Laptop incluso e opzionale e disattivato per impostazione predefinita. Puo aggiungere temperature e RPM ventola Framework quando Framework Control e installato e in esecuzione.

Per sviluppatori, l albero sorgente GitHub include Docs\\Plug-In-development.md.

Flusso controllo ventole

Aprire Opzioni > Controllo ventole... o Ctrl+L.

1. Scegliere un controllo. 2. Opzionale: dare un nome e premere Alt+L. 3. Inserire percentuale 0-100. 4. Alt+M applica. 5. Alt+A riporta la selezione ad automatico. 6. Alt+R riporta tutti i visibili ad automatico.

I pulsanti globali agiscono solo sui controlli visibili. Intestazioni ferme o vuote sono nascoste finche Mostra ferme non e attivo. I nomi sono salvati nella configurazione specifica del computer in Config e cambiano solo l'aspetto in Sensor Readout.

Curve ventole

Aprire le curve da Opzioni > Curve ventole... o dalla finestra Controllo ventole.

Ogni curva sceglie un controllo ventola scrivibile e una lettura temperatura. Sensor Readout imposta la ventola alla percentuale bassa alla temperatura bassa, aumenta tra i punti basso e alto e usa la percentuale di emergenza quando viene raggiunta la temperatura di emergenza. Le curve dipendono da un controllo scrivibile esposto da LibreHardwareMonitor; se l hardware espone solo RPM, Sensor Readout puo leggere la ventola ma non controllarla.

Profili ventole

Preferenze > Profili ventole consente di creare gruppi nominati di azioni ventola. Il layout corrisponde alla scheda Hotkeys: creare un profilo, assegnare un hotkey globale opzionale, aggiungere controlli e scegliere percentuale manuale o automatico/predefinito per ogni ventola.

I profili ventole possono anche essere marcati come interruttori. Con questa opzione, la prima pressione applica il profilo e la seconda riporta le ventole di quel profilo ad automatico/predefinito.

  • Uso quotidiano: ventole case silenziose e CPU automatica.
  • Gioco o rendering: aumentare case, CPU e GPU prima di un carico pesante.
  • Ritorno automatico: un profilo che rimette tutti i controlli scrivibili in automatico/predefinito.

I profili sono specifici del computer perche i controlli dipendono dall hardware attuale. Le nuove installazioni partono con profili iniziali vuoti per uso quotidiano, carichi pesanti e ritorno automatico; aggiungere le ventole corrette, rinominarli o eliminarli. Una curva attiva puo regolare di nuovo la stessa ventola al prossimo aggiornamento.

Esempi di configurazione

  • Hotkey stato rapido: uso CPU, temperatura CPU, temperatura GPU e velocita rete importanti.
  • Attivita disco: lettura e scrittura delle unita importanti, con doppia pressione per copiare.
  • Desktop silenzioso: curva CPU, profilo case silenzioso e allarme temperatura.
  • Diagnosi USB: aprire USB, premere Invio su un dispositivo, scegliere campi utili e copiarli.

Diagnostica

Usare Aiuto > Esegui diagnostica... o premere Alt+F1 quando serve raccogliere informazioni di supporto. Sensor Readout crea un file ZIP in Reports con il nome del computer nel nome del file, apre la cartella al termine e rimuove la cartella temporanea dopo la creazione dello ZIP.

Lo ZIP diagnostico contiene un report TXT, un report HTML, un log di debug, un riepilogo diagnostico e informazioni sui tempi. L esecuzione prova brevemente i controlli ventola scrivibili al 100%, poi ripristina ogni ventola allo stato precedente manuale, automatico/predefinito o curva ventola.

La diagnostica puo pronunciare l avanzamento e riprodurre suoni di inizio/fine. Configurare questa opzione in Opzioni > Preferenze > Avvio. La diagnostica da riga di comando usa le stesse preferenze salvo l uso di --diagnostics-quiet, --no-diagnostics-speech o --no-diagnostics-sounds.

Per test automatici, eseguire Sensor Readout.exe --diagnostics [percorso]. Se [percorso] e una cartella, lo ZIP viene creato li. Senza percorso, Sensor Readout crea lo ZIP in Reports.

Report

I report sono utili in due situazioni: conservare un istantanea del proprio computer, oppure inviarla a qualcun altro per aprirla dentro Sensor Readout.

Salvare un report

Premere Ctrl+S o usare File > Salva report....

La finestra di salvataggio offre report TXT e HTML. TXT raggruppa per tipo di lettura e dispositivo, cosi i nomi hardware lunghi vengono scritti una volta come intestazioni. HTML usa tabelle con sensore, valore e fonte.

Il primo salvataggio crea la cartella Reports se non esiste. Per condividere un report, inviare il file .html o .txt salvato. HTML di solito e piu facile da leggere in un browser; TXT e comodo da incollare nei messaggi. Se serve un pacchetto di supporto piu completo, usare Aiuto > Esegui diagnostica...; la diagnostica include entrambi i formati, log e riepilogo in uno ZIP.

Aprire il report di un altro computer

Premere Ctrl+O o usare File > Apri report... per caricare un report TXT o HTML salvato da Sensor Readout.

Il report si apre come istantanea statica nella normale vista categorie e albero. Cosi e possibile esaminare il computer di un altra persona come se fosse la vista corrente. La modalita report statico non aggiorna valori live, non esegue allarmi e non controlla hardware. Mostra solo i dati salvati nel report.

I dettagli e i comandi di copia funzionano ancora quando il report contiene i campi necessari. Questo e utile quando qualcuno invia dati USB, audio, schermo, rete, ventole, batteria o SMART e si vogliono controllare righe specifiche senza leggere manualmente tutto il file.

Usare File > Torna alle letture live o Ctrl+R per tornare al computer corrente. I report salvati da Sensor Readout 2.0 o versioni successive includono un istantanea leggibile dall app per la migliore importazione. I vecchi report TXT e HTML vengono aperti nel miglior modo possibile.

File

Build portable:

  • Sensor Readout.exe: applicazione principale.
  • Install-Prerequisites.cmd e Install-Prerequisites.ps1: strumenti opzionali per .NET Framework, PawnIO e diagnostica.

Cartelle usate da Sensor Readout:

  • Config\Shared.json: preferenze portabili condivise tra computer, come lingua, aggiornamento, unita temperatura, suoni, messaggio di avvio, area notifica, aggiornamenti e hotkey globali.
  • Config\<ComputerName>.json: preferenze specifiche del computer, come avvio con Windows, letture area notifica, hotkey parlate, allarmi, letture nascoste, nomi ventola, controllo ventole, nomi parlati personalizzati, logging e avvisi primo avvio.
  • Logs\<ComputerName>.log: diagnostica opzionale, azioni ventola, hotkey e messaggi screen reader.
  • Reports: cartella predefinita per report da riga di comando, primo dialogo Salva o Aiuto > Esegui diagnostica....
  • Langs: file lingua modificabili.
  • Docs: manuali HTML inclusi.
  • Sounds: WAV inclusi come SR01.wav e WAV aggiunti dall'utente.
  • Plug-Ins: plug-in hardware opzionali. I plug-in inclusi, come Framework Laptop e i supporti sperimentali HP Hardware, Dell Latitude e Asus ROG/TUF, restano disattivati finché non vengono abilitati in Preferenze. Il plug-in Asus include il proprio avviso GPL.
  • Data\usb.ids: dati USB produttore/prodotto.
  • Data\oui.csv: dati MAC/OUI per adattatori di rete.

File lingua

  • Mettere file .txt o .ini in Langs.
  • Al primo avvio Sensor Readout sceglie la lingua Windows se presente, altrimenti inglese.
  • Usare key=value; # e ; sono commenti.
  • language.name e il nome visualizzato.
  • manual.file=README-xx.html indica il manuale per F1. Solo nome file, senza percorso. Cerca in Docs, poi usa inglese.
  • number.decimalSeparator=, imposta il separatore decimale predefinito.
  • La cartella viene controllata circa ogni 15 secondi.
  • Langs\English.txt e lingua principale e modello.
  • Il separatore decimale puo essere cambiato anche in Preferenze.

La voce screen reader opzionale usa Tolk.dll, SAAPI64.dll e DLL di supporto Tolk se screen reader e in esecuzione. All'avvio minimizzato puo parlare un messaggio configurabile. L'editor lingua consente modifiche senza altro editor.

Note

Il supporto ventole dipende da LibreHardwareMonitor, PawnIO, chip sensore e BIOS. Alcuni sistemi danno letture ma non controlli scrivibili. L'accesso Super I/O scheda madre spesso richiede amministratore.

Se mancano temperatura CPU o carico CPU, puo aiutare installare e avviare Core Temp. Sensor Readout puo leggere i dati in memoria condivisa di Core Temp quando Core Temp e gia in esecuzione. Core Temp e opzionale e viene usato solo come fonte alternativa per letture CPU. Usare Aiuto > Installa supporto Core Temp... per aprire la pagina ufficiale Core Temp o avviare l installazione con winget o Chocolatey quando questi strumenti sono disponibili.

Se i controlli ventola sembrano mancanti, aprire Opzioni > Controllo ventole... e attivare Mostra ferme. Alcune schede segnalano connettori controllabili come fermi o indefiniti finche non iniziano a girare; questa opzione li rende visibili per prova.

Portatili Framework

Gli utenti di portatili Framework che vogliono letture specifiche di temperatura e RPM ventola devono prima aggiornare il BIOS da Framework BIOS and Drivers Downloads, poi installare ed eseguire Framework Control. Se Framework Control mostra controlli ventola e temperatura nell interfaccia browser, aprire Opzioni > Preferenze > Plug-Ins, abilitare il plug-in Framework Laptop e chiudere Preferenze. Sensor Readout puo quindi leggere la sua API locale e mostrare righe temperatura e ventola Framework Control. Sensor Readout legge solo questa API locale; non installa Framework Control, non cambia impostazioni ventola tramite quell API e non aggiorna firmware.
