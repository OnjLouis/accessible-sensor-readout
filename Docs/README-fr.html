Sensor Readout

Version actuelle: 3.7.0.

Sensor Readout est un utilitaire Windows pour lire les capteurs materiels et controler les ventilateurs compatibles, avec une interface concue d'abord pour le clavier et les lecteurs d'ecran.

Les categories principales sont a gauche, les mesures groupees par appareil ou usage sont dans une arborescence a droite, et les commandes communes sont dans une barre de menus standard.

Sensor Readout est disponible sur GitHub.

Pourquoi Sensor Readout existe

Toutes ces informations sur votre propre ordinateur, autrefois difficiles a trouver, cachees derriere le Gestionnaire de peripheriques ou enfermees dans des apps couteuses ou inaccessibles, devraient etre plus faciles a atteindre.

Sensor Readout vise a reunir autant de ces informations que possible dans une app gratuite, facile a utiliser, portable ou installable, pour vous eviter de les chercher partout. J en avais assez de l etat de l accessibilite du cote technique de l informatique Windows, alors j ai voulu obtenir un outil que j aurais moi-meme envie d utiliser. Manifestement, beaucoup d autres personnes le souhaitaient aussi.

J espere que vous decouvrirez autant de choses sur votre ordinateur avec cette app que j en ai decouvert sur le mien, et j espere que ce que vous trouverez vous surprendra agreablement.

Mon souhait est que Sensor Readout devienne la reference de sa categorie: un outil pense d abord pour l accessibilite, que les personnes aveugles et malvoyantes travaillant dans le support technique puissent donner a leurs clients afin qu ils generent des rapports et obtiennent de l aide pour resoudre les problemes. Je vise l excellence, et j espere que vous le ressentirez en l utilisant.

Fonctions

  • Lit temperatures, vitesse des ventilateurs, etat et capacite du stockage, ainsi que certains compteurs materiels.
  • Affiche une categorie Peripheriques avec inventaire PnP Windows, dont peripheriques PCI/systeme, controleurs de stockage, peripheriques d entree, Bluetooth, imprimantes, peripheriques de securite et entrees croisees concises pour le materiel USB, audio, affichage et reseau.
  • Affiche une page Performances/Vue d'ensemble pour duree de fonctionnement, BIOS, GPU, CPU, memoire et activite lecture/ecriture des disques.
  • Ouvre l'interface principale immediatement pendant que la premiere actualisation continue en arriere-plan.
  • Affiche les cartes reseau avec etat, adresses IP, vitesse de lien, debits et trafic total.
  • Affiche les adresses MAC et les fabricants OUI des cartes reseau lorsque la base OUI incluse connait le prefixe.
  • Affiche une categorie USB pour les peripheriques connectes, hubs, controleurs, vitesse de connexion, puissance lorsque Windows l expose, lettres de lecteurs, etat de retrait securise et champs detailles copiables.
  • Affiche charge de batterie, etat, capacite, sante, cycles, tension et puissance lorsque Windows les fournit.
  • Prend en charge des courbes simples de ventilateur qui reglent un controle a partir d une temperature choisie.
  • Prend en charge les profils de ventilateur qui appliquent plusieurs controles a la fois, avec raccourcis globaux optionnels et retour automatique optionnel au second appui.
  • Ajoute des liens vers la page du projet depuis README, Aide et A propos.
  • Utilise les bibliotheques LibreHardwareMonitor incluses.
  • Utilise le pilote PawnIO pour les capteurs de carte mere et le controle de ventilateurs lorsque le materiel le permet.
  • Permet de nommer les connecteurs de ventilateur.
  • Masque par defaut les connecteurs arretes ou non equipes.
  • Applique un pourcentage manuel a un ventilateur selectionne ou a tous les ventilateurs visibles.
  • Remet un ventilateur ou tous les ventilateurs en automatique/defaut.
  • Enregistre des rapports TXT ou HTML.
  • Ouvre directement les rapports enregistres, y compris les rapports TXT ou HTML envoyes dans un fichier ZIP.
  • Permet de configurer l'actualisation automatique et utilise 5 secondes par defaut.
  • Peut demarrer avec Windows et se reduire dans la zone de notification.
  • Utilise Config\Shared.json pour les preferences portables qui doivent suivre l application entre ordinateurs, tout en gardant la configuration propre au materiel dans Config\<ComputerName>.json.
  • Enregistre les preferences au moment du changement, pour mieux conserver hotkeys, zone de notification et elements masques.
  • Ecrit les journaux de diagnostic dans Logs\<ComputerName>.log quand la journalisation est activee.
  • Affiche les temperatures en Celsius, Fahrenheit, Celsius puis Fahrenheit ou Fahrenheit puis Celsius.
  • Prend en charge des hotkeys globaux pour afficher/masquer et lire l'etat de la zone de notification.
  • Peut parler via lecteur d'ecran avec les DLL bibliotheque de lecteur d'ecran Tolk 64 bits incluses.
  • Utilise des fichiers de langue simples dans Langs.
  • La journalisation est desactivee par defaut et peut etre activee pour le depannage.
  • Peut afficher des mesures choisies dans l'info-bulle de la zone de notification.

Prerequis

  • Windows 10 ou Windows 11, 64 bits.
  • Microsoft .NET Framework 4.8 ou plus recent.
  • Pilote PawnIO pour les capteurs de carte mere et le controle des ventilateurs.
  • Droits administrateur pour lire beaucoup de capteurs de carte mere ou controler les ventilateurs.

LibreHardwareMonitor n'a pas besoin de tourner comme application separee car les bibliotheques necessaires sont fournies. L'application autonome peut rester utile pour diagnostiquer.

Programme d installation des prerequis

Executez d abord Install-Prerequisites.cmd depuis ce dossier. Il appelle Install-Prerequisites.ps1, demande les droits administrateur si necessaire et installe PawnIO. Il essaie winget, puis Chocolatey s il est deja installe, puis telecharge la version officielle PawnIO.Setup depuis GitHub si aucun gestionnaire de paquets n est disponible. Quand winget existe, il peut aussi installer .NET Framework Runtime.

Install-Prerequisites.cmd

Pour installer aussi LibreHardwareMonitor autonome avec winget pour le depannage, executez :

.\Install-Prerequisites.ps1 -IncludeLibreHardwareMonitor

Commandes d installation manuelle

Si le programme d installation des prerequis ne fonctionne pas sur votre systeme, ouvrez Windows Terminal ou PowerShell comme administrateur et executez ces commandes :

winget install --id Microsoft.DotNet.Framework.Runtime -e --source winget

winget install --id namazso.PawnIO -e --source winget

Outil optionnel de diagnostic :

winget install --id LibreHardwareMonitor.LibreHardwareMonitor -e --source winget

Premiers pas

1. Lancez Sensor Readout.exe.

2. Acceptez l'invite administrateur Windows.

3. Si PawnIO manque, Sensor Readout propose l'installateur de prerequis.

4. Appuyez sur F5 si les valeurs chargent encore.

5. Ouvrez Options > Controle des ventilateurs... ou appuyez sur Ctrl+L.

Si des ventilateurs ou controles de carte mere manquent, verifiez PawnIO et le lancement comme administrateur. L'installateur est disponible plus tard depuis Aide > Installer les prerequis....

Preferences

Utilisez Options > Preferences ou Ctrl+,. La boite garde le dernier onglet utilise pendant la session.

General (Ctrl+1)

Langue, actualisation, intervalle, icone de notification, unite de temperature, separateur decimal, niveau de journalisation, verification des mises a jour et mesures de notification.

Demarrage et installation (Ctrl+2)

Installation sur ce PC, demarrage Windows, demarrage reduit, message vocal de demarrage, sons de debut/fin et retour de diagnostic.

Installer sur ce PC... copie le dossier portable actuel vers le dossier des programmes Windows de cet utilisateur, peut creer un raccourci sur le bureau, ferme l ancienne copie et lance la copie installee. Si le demarrage Windows est actif, le raccourci de demarrage pointe ensuite vers la copie installee.

Lorsque Sensor Readout s execute deja depuis l emplacement installe, ce bouton devient Desinstaller de ce PC.... La desinstallation supprime les fichiers installes et les raccourcis, mais conserve Config, Logs et Reports. Le meme onglet contient aussi une case de raccourci bureau pour l ajouter ou le retirer sans reinstaller.

Si Sensor Readout redemarre apres une installation ou une mise a jour et que la fenetre principale n apparait pas, verifiez si Demarrer reduit dans la zone de notification est active. Windows peut aussi masquer l icone tant que vous ne l autorisez pas dans les parametres de zone de notification ou de barre d etat systeme. En affichant l icone Sensor Readout, vous pourrez ouvrir la fenetre principale depuis cette icone.

Raccourcis (Ctrl+3)

Afficher/masquer global, annoncer la zone de notification, profils de raccourcis parles, copie par double pression et noms parles courts.

Profils ventilateur (Ctrl+4)

Groupes d actions ventilateur avec raccourci optionnel, mode bascule, message vocal et son.

Alarmes (Ctrl+5)

Surveiller des mesures, choisir seuil, unite, delai, parole et son.

Plug-Ins (Ctrl+6)

Voir, decrire, importer et activer les plug-ins materiels installes.

Elements masques (Ctrl+7)

Restaurer les mesures ou groupes masques dans la fenetre principale.

L onglet inclut aussi la preference de confirmation d installation des mises a jour, pour reactiver l avertissement de securite apres l avoir masque.

Editeur de langue (Ctrl+8)

Modifier les fichiers de langue ou en creer un depuis l anglais. Les chaines manquantes retombent sur l anglais.

Quand l etat de notification est active, reduire Sensor Readout le retire de la barre des taches et de Alt+Tab. Rouvrez-le depuis l icone. Alt+F4 quitte l application.

Les mesures de la zone de notification passent d une liste disponible a une liste d ordre. La limite est de quatre car les info-bulles Windows sont courtes. Une mesure n apparait que dans une liste. Utilisez les boutons ou Ctrl+fleches. Les elements affichent appareil, nom et categorie, par exemple Ethernet - Rx: Reseau.

L onglet Raccourcis peut importer des profils de raccourcis parles depuis une autre configuration de machine. Les profils importes ne gardent pas leurs anciennes touches, afin de ne pas voler des touches deja utilisees sur cette machine. Les lectures sont conservees seulement quand Sensor Readout peut les faire correspondre sans risque sur l ordinateur actuel.

Lecture des capteurs

Le panneau de mesures est une arborescence. Temperatures, Ventilateurs, SMART, Performances et Reseau groupent par appareil ou usage puis par valeur.

La liste de gauche change seulement la vue; elle n'active ni ne desactive les appareils. Sensor Readout demarre sur Performance/Vue d'ensemble. Performances regroupe CPU, memoire et activite disque. Reseau affiche etat, IP, vitesse, debits et compteurs.

Les pourcentages sont aussi exposes dans la barre de progression inferieure pour les lecteurs d'ecran. Utilisez Edition, touche Applications ou clic droit pour copier, renommer un ventilateur ou masquer des valeurs.

Alarmes et sons

Preferences > Alarmes permet de creer des alarmes de mesures. Choisissez une mesure, Superieur ou egal, Inferieur ou egal, ou Egal, puis le seuil et le delai. Chaque alarme peut parler via lecteur d'ecran, jouer un WAV, ou les deux.

Preferences > Demarrage et installation contient les sons optionnels de demarrage et fermeture. Sensor Readout charge les WAV depuis le dossier Sounds. Les sons fournis utilisent des noms neutres comme SR01.wav, SR02.wav, etc.; les sons ajoutes par l'utilisateur peuvent avoir n'importe quel nom de fichier .wav normal. Tout son de ce dossier peut etre choisi partout ou Sensor Readout propose un son, notamment alarmes, demarrage ou fermeture, diagnostics, alertes de mise a jour et actions de profils de ventilateur.

Plug-Ins

Preferences > Plug-Ins liste les plug-ins materiels installes et decrit leur role. Les plug-ins sont du code de confiance; n activez que ceux venant de personnes ou projets fiables.

Les plug-ins installes commencent desactives. Importer un ZIP copie le plug-in dans Plug-Ins, mais ne l active pas automatiquement.

Quand un plug-in active declare des pages d aide liees, ces liens apparaissent dans le menu Aide. Par exemple, les plug-ins portables fournis peuvent afficher les pages Dell Command | Monitor, G-Helper, HP Support Assistant, OMEN Gaming Hub, Framework BIOS and Drivers ou Framework Control.

Le plug-in Framework Laptop fourni est optionnel et desactive par defaut. Il peut ajouter les temperatures et RPM de ventilateur Framework quand Framework Control est installe et lance.

Pour les developpeurs, l arbre source GitHub contient Docs\\Plug-In-development.md.

Utilisez Edition > Rechercher une lecture... ou F3 pour chercher dans toutes les categories. La recherche se reduit pendant la frappe. Tab va a la liste des resultats, Entree va a la mesure selectionnee, Alt+L efface la recherche, et Esc ou Fermer revient a la fenetre principale.

Controle des ventilateurs

Ouvrez Options > Controle des ventilateurs... ou Ctrl+L.

1. Choisissez un controle. 2. Optionnel : nommez-le et appuyez sur Alt+L. 3. Entrez un pourcentage de 0 a 100. 4. Alt+M applique. 5. Alt+A remet la selection en automatique. 6. Alt+R remet tous les visibles en automatique.

Les boutons globaux n'affectent que les controles visibles. Les connecteurs arretes ou vides sont masques tant que Afficher arretes n'est pas coche. Les noms sont gardes dans la configuration propre a l'ordinateur dans Config et ne changent que l'affichage.

Courbes de ventilateur

Ouvrez les courbes depuis Options > Courbes de ventilateur... ou depuis la fenetre de controle des ventilateurs.

Chaque courbe choisit un controle de ventilateur modifiable et une temperature. Sensor Readout applique le pourcentage bas a la temperature basse, augmente progressivement entre les points bas et haut, puis utilise le pourcentage d urgence lorsque la temperature d urgence est atteinte. Les courbes dependent d un controle modifiable expose par LibreHardwareMonitor; si le materiel expose seulement les RPM, Sensor Readout peut lire le ventilateur mais pas le controler.

Profils de ventilateurs

Preferences > Profils de ventilateurs permet de creer des groupes nommes d actions. La presentation reprend l onglet Hotkeys : creer un profil, attribuer un raccourci global optionnel, ajouter des controles, puis choisir un pourcentage manuel ou automatique/par defaut pour chaque ventilateur.

Les profils de ventilateur peuvent aussi etre marques comme bascules. Avec cette option, le premier appui applique le profil et le second remet les ventilateurs de ce profil en automatique/par defaut.

  • Quotidien : ventilateurs de boitier silencieux et CPU automatique.
  • Jeu ou rendu : augmenter boitier, CPU et GPU avant une lourde charge.
  • Retour automatique : un profil qui remet tous les controles modifiables en automatique/par defaut.

Les profils sont propres a chaque machine, car les controles dependent du materiel actuel. Les nouvelles installations commencent avec des profils de depart vides pour usage quotidien, charge elevee et retour automatique; ajoutez les ventilateurs adaptes, renommez-les ou supprimez ces profils. Une courbe active peut reajuster le meme ventilateur a la prochaine actualisation.

Exemples de configuration

  • Raccourci d etat rapide : charge CPU, temperature CPU, temperature GPU et debits reseau importants.
  • Activite disque : debits lecture/ecriture des lecteurs importants, avec double pression pour copier.
  • Bureau silencieux : courbe CPU, profil silencieux de boitier et alarme de temperature.
  • Diagnostic USB : ouvrir USB, appuyer sur Entree sur un peripherique, selectionner des champs utiles et copier.

Diagnostics

Utilisez Aide > Executer les diagnostics... ou Alt+F1 lorsque vous devez collecter des informations de support. Sensor Readout cree un fichier ZIP dans Reports avec le nom de l ordinateur dans le nom du fichier, ouvre le dossier a la fin et supprime le dossier temporaire apres la creation du ZIP.

Le ZIP de diagnostic contient un rapport TXT, un rapport HTML, un journal de debogage, un resume de diagnostic et des informations de duree. Le test essaie brievement les controles de ventilateur modifiables a 100%, puis restaure chaque ventilateur dans son etat precedent manuel, automatique/par defaut ou courbe de ventilateur.

Les diagnostics peuvent annoncer la progression et jouer des sons de debut/fin. Configurez cela dans Options > Preferences > Demarrage. Les diagnostics en ligne de commande utilisent les memes preferences sauf si vous passez --diagnostics-quiet, --no-diagnostics-speech ou --no-diagnostics-sounds.

Pour les tests sans surveillance, lancez Sensor Readout.exe --diagnostics [chemin]. Si [chemin] est un dossier, le ZIP est cree a cet endroit. Sans chemin, Sensor Readout cree le ZIP dans Reports.

Rapports

Les rapports servent a deux choses : garder un instantane de votre propre machine, ou envoyer cet instantane a quelqu un d autre pour qu il puisse l examiner dans Sensor Readout.

Enregistrer un rapport

Appuyez sur Ctrl+S ou utilisez Fichier > Enregistrer le rapport....

La fenetre d enregistrement propose des rapports TXT et HTML. Le TXT groupe les valeurs par type de mesure et par appareil, afin que les longs noms materiels ne soient pas repetes sur chaque ligne. Le HTML utilise des tableaux avec capteur, valeur et source.

Le premier enregistrement cree le dossier Reports s il n existe pas. Pour partager un rapport, envoyez le fichier .html ou .txt enregistre. Le HTML est generalement plus facile a lire dans un navigateur; le TXT est pratique pour coller dans un message. Si quelqu un a besoin d un paquet de support complet, utilisez plutot Aide > Executer les diagnostics...; les diagnostics incluent les deux formats, les journaux et un resume dans un ZIP.

Ouvrir le rapport d un autre ordinateur

Appuyez sur Ctrl+O ou utilisez Fichier > Ouvrir un rapport... pour charger un rapport TXT ou HTML Sensor Readout. Si quelqu un envoie un rapport dans un fichier ZIP, ouvrez directement le ZIP; Sensor Readout utilisera le premier rapport lisible trouve.

Le rapport s ouvre comme un instantane statique dans la presentation normale par categories et arborescence. Vous pouvez ainsi examiner la machine d une autre personne comme si c etait votre vue actuelle. Le mode rapport statique ne met pas les valeurs a jour, ne lance pas les alarmes et ne controle aucun materiel. Il affiche seulement les donnees enregistrees dans le rapport.

Les details et commandes de copie fonctionnent encore lorsque le rapport contient les champs necessaires. C est utile si quelqu un envoie des donnees USB, audio, affichage, reseau, ventilateur, batterie ou SMART et que vous voulez verifier des lignes precises sans lire tout le fichier a la main.

Utilisez Fichier > Revenir aux lectures en direct ou Ctrl+R pour revenir a l ordinateur actuel. Les rapports HTML crees par Sensor Readout 2.0 ou plus recent contiennent un instantane masque lisible par l application pour la meilleure importation. Les rapports TXT restent en texte simple et sont ouverts au mieux.

Transfert des parametres

Utilisez Fichier > Exporter les parametres et profils... ou Ctrl+E pour creer un paquet de parametres. Sensor Readout affiche d abord une liste de controle, afin que vous puissiez choisir exactement quoi inclure. Utilisez Fichier > Importer les parametres et profils... ou Ctrl+Maj+I pour les importer. L importation affiche le meme type de liste, limitee aux sections trouvees dans le paquet choisi. Les mesures portables comme la memoire, le CPU, le temps actif du systeme et de nombreuses valeurs de synthese sont associees lorsque c est possible. Les elements propres au materiel sont importes avec prudence : les raccourcis globaux restent vides, les actions de profils de ventilateur ne sont pas liees, les courbes de ventilateur restent desactivees jusqu au choix d une commande de ventilateur, et les alarmes restent desactivees pour verification.

Fichiers

Version portable :

  • Sensor Readout.exe : application principale.
  • Install-Prerequisites.cmd et Install-Prerequisites.ps1 : aides optionnelles pour .NET Framework, PawnIO et le diagnostic.

Dossiers utilises par Sensor Readout :

  • Config\Shared.json : preferences portables partagees entre ordinateurs, comme langue, actualisation, unite temperature, sons, message de demarrage, notification, mises a jour et hotkeys globaux.
  • Config\<ComputerName>.json : preferences propres a la machine, comme demarrage Windows, mesures de notification, raccourcis parles, alarmes, mesures masquees, noms de ventilateurs, controle des ventilateurs, noms parles personnalises, journalisation et alertes de premier lancement.
  • Logs\<ComputerName>.log : diagnostics optionnels, actions ventilateur, hotkeys et messages lecteur d'ecran.
  • Reports : dossier par defaut pour les rapports de ligne de commande, du premier dialogue Enregistrer ou de Aide > Executer les diagnostics....
  • Langs : fichiers de langue modifiables.
  • Docs : manuels HTML fournis.
  • Sounds : sons WAV fournis comme SR01.wav et WAV ajoutes par l'utilisateur.
  • Plug-Ins : plug-ins materiels optionnels. Les plug-ins fournis, comme Framework Laptop et HP Hardware Support experimental, restent desactives jusqu'a activation dans Preferences.

Fichiers de langue

  • Placez les fichiers .txt ou .ini dans Langs.
  • Au premier lancement, Sensor Readout choisit la langue Windows si disponible, sinon l'anglais.
  • Utilisez key=value; # et ; sont des commentaires.
  • language.name est le nom affiche.
  • manual.file=README-xx.html indique le manuel de F1. Nom de fichier seulement, pas de chemin. Recherche dans Docs, sinon anglais.
  • number.decimalSeparator=, definit le separateur decimal par defaut.
  • Le dossier est verifie environ toutes les 15 secondes.
  • Langs\English.txt est la langue principale et le modele.
  • Le separateur decimal peut aussi etre change dans Preferences.

La voix lecteur d'ecran optionnelle utilise Tolk.dll, SAAPI64.dll et DLL de support Tolk si lecteur d'ecran fonctionne. Au demarrage reduit, un message configurable peut etre prononce. L'editeur de langue permet de modifier sans autre editeur.

Notes

Le support des ventilateurs depend de LibreHardwareMonitor, PawnIO, du controleur de capteurs et du BIOS. Certains systemes donnent des mesures mais pas de controles modifiables. L'acces Super I/O de carte mere exige souvent l'administrateur.

Si la temperature CPU ou la charge CPU manque, installer et lancer Core Temp peut aider. Sensor Readout peut lire les donnees en memoire partagee de Core Temp quand Core Temp est deja lance. Core Temp est optionnel et sert seulement de source de repli pour les mesures CPU.

Si les controles de ventilateur semblent absents, ouvrez Options > Controle des ventilateurs... et activez Afficher les arretes. Certaines cartes signalent des connecteurs controlables comme arretes ou indefinis jusqu'a ce qu'ils tournent; cette option les rend visibles pour tester.

Plug-ins de marque de portable

Les utilisateurs de portables Framework, Dell, Lenovo, Asus, HP, OMEN et Victus peuvent obtenir une meilleure visibilite des temperatures ou ventilateurs en activant le plug-in materiel correspondant dans Options > Preferences > Plug-Ins. Les plug-ins restent desactives tant que vous ne les activez pas, et les changements s appliquent apres la fermeture des Preferences.

Les utilisateurs Framework qui veulent les RPM du ventilateur et des temperatures propres au modele doivent d abord mettre a jour le BIOS depuis Framework BIOS and Drivers Downloads, puis installer et lancer Framework Control. Si Framework Control affiche les controles de ventilateur et de temperature dans son interface web, activez le plug-in Framework Laptop dans Sensor Readout.

Les plug-ins Dell, Lenovo, Asus, HP, OMEN et Victus ne garantissent pas les lectures de ventilateur sur chaque modele, mais ils valent la peine d etre essayes avant d envoyer des diagnostics ou de supposer que le portable n expose aucune donnee de capteur supplementaire.

Des outils optionnels du fabricant peuvent aussi aider a exposer ou verifier les donnees propres au portable. Les utilisateurs Dell peuvent essayer Dell Command | Monitor. Les utilisateurs Lenovo peuvent essayer Lenovo Vantage ou Lenovo System Update. Les utilisateurs Asus peuvent essayer G-Helper. Les utilisateurs HP, OMEN et Victus peuvent essayer HP Support Assistant et OMEN Gaming Hub. Ces outils ne font pas partie de Sensor Readout; utilisez les pages du fabricant ou du projet et installez seulement ce qui a du sens pour votre machine.

Sensor Readout lit seulement ces chemins de support optionnels sauf indication contraire d un plug-in; il ne flashe pas le firmware et ne remplace pas les outils du fabricant.
