# Sensor Readout Source File Map

This table explains the current C# source split used by Sensor Readout. It is intended for contributors and anyone who wants to understand where a feature lives before reading the code. Keep it updated whenever files are added, removed, renamed, or given a materially different responsibility.

| Source file | Main responsibility | What it does | Notes |
| --- | --- | --- | --- |
| `AssemblyInfo.cs` | Assembly metadata | Sets the application assembly title and version numbers compiled into the executable. | Small metadata file; version should match the public release. |
| `Models.cs` | Shared data models | Defines the plain data objects used across the app: settings, sensor rows, tray selections, fan labels, spoken hotkey profiles, fan profiles, alarms, plug-in preferences, plug-in help links, update info, and language choices. | Keeps cross-feature data contracts away from UI code. |
| `MacVendorDatabase.cs` | MAC/OUI vendor lookup | Loads the bundled OUI vendor database from `Data` and resolves MAC address or OUI-style prefixes to vendor names. | Used by the Network category and USB network/storage details. |
| `NativeInterop.cs` | Windows and screen-reader interop | Contains P/Invoke declarations and small wrappers for Windows APIs, global hotkeys, shutdown handling, memory-mapped Core Temp data, Tolk screen-reader speech, and related native structures. | This is the boundary between managed C# and native DLL/Windows calls. |
| `Program.cs` | Application startup and command line | Parses command-line options, handles single-instance and close behavior, report-only modes, diagnostics-only mode, logging override, TLS setup, crash logging/restart handoff, and starts the WinForms message loop. | Entry point for `Sensor Readout.exe`. |
| `PreferencesForm.cs` | Preferences window | Builds and manages Preferences: General, Startup and Install, Hotkeys, Fan profiles, Alarms, Plug-Ins, Hidden items, and Language editor tabs. Handles immediate saving, accessible labels, list ordering/search, alarm editing, update preferences, desktop/startup shortcut choices, plug-in ZIP import, sound preview, install/uninstall request wiring, and keyboard navigation inside Preferences. | Largest file because the preferences dialog has many controls and workflows. |
| `SensorReadoutForm.cs` | Main window shell | Defines the main form fields and constructor, creates the primary UI layout, menu structure, category list, tree view, status/progress area, timers, and high-level event wiring. | The central partial class that the feature-specific files extend. |
| `SensorReadoutForm.Alarms.cs` | Alarms and sounds runtime | Evaluates alarm rules against current readings, triggers screen-reader speech and WAV playback, flashes the tray icon, plays startup/shutdown sounds, and loads sound file names from the `Sounds` folder. | Uses WAV playback through .NET `SoundPlayer`. |
| `SensorReadoutForm.Audio.cs` | Audio device inventory | Reads Windows sound devices and audio endpoint data, including vendor, status, playback/recording direction, and default channel/sample-rate/bit-depth format where exposed. | Used by the Audio category and reports. |
| `SensorReadoutForm.Battery.cs` | Battery readings | Reads battery charge, status, capacity, health, cycle count, voltage, and power-rate information where Windows exposes it. | Used by laptops and other systems with batteries. |
| `SensorReadoutForm.Dispose.cs` | Cleanup | Disposes timers, tray icon, hardware computer objects, hidden windows, and other resources when the app exits. | Keeps shutdown cleanup separate from feature logic. |
| `SensorReadoutForm.Display.cs` | Display inventory | Reads graphics adapters and monitor identity data, including adapter memory, current resolution, driver information, monitor vendor, product code, serial, and manufacture date where exposed. | Used by the Display category and reports. |
| `SensorReadoutForm.Diagnostics.cs` | One-click diagnostics | Runs Help-menu diagnostics, temporarily enables debug-level logging in memory, creates TXT/HTML reports, exercises writable fan controls, restores fan/manual/curve state, writes a summary, zips the diagnostic bundle with the computer name in the file name, removes the temporary staging folder, and opens the Reports folder. | Designed for support cases where users need to send one ZIP instead of several separate files. |
| `SensorReadoutForm.FanControls.cs` | Fan controls dialog | Builds and manages the Fan Controls dialog, including fan selection, friendly labels, manual percentages, automatic/default control, show-stopped behavior, and visible fan command buttons. | Fan control depends on LibreHardwareMonitor/PawnIO and writable controls exposed by hardware. |
| `SensorReadoutForm.FanCurves.cs` | Fan curves | Builds and manages the Fan Curves dialog, stores curve definitions, and applies enabled curves by mapping selected temperature readings to writable fan control percentages. | Curve support still depends on hardware exposing writable fan controls. |
| `SensorReadoutForm.Formatting.cs` | Display formatting | Formats numbers, temperatures, bytes, rates, percentages, uptime, fan display text, speech labels, and selected meter values. | Keeps text/value presentation consistent across UI, tray, reports, hotkeys, and alarms. |
| `SensorReadoutForm.HotkeysTrayHelp.cs` | Hotkeys, tray, help, and screen-reader speech | Registers and handles global hotkeys, show/hide behavior, spoken tray status, custom spoken hotkey profiles, double-press clipboard copy, tray icon behavior, Help/About/Donate/Contact links, version history, dynamic enabled plug-in help links, and F1 manual opening. | Main accessibility and notification-area interaction layer. |
| `SensorReadoutForm.Install.cs` | Install-to-PC and uninstall flow | Detects whether the app is running from the local install folder, copies the current app folder into the normal Windows programs folder for the current user, manages desktop and startup shortcuts, closes the old copy, relaunches from the installed location, and removes installed app files while preserving Config/Logs/Reports during uninstall. | Used by Preferences > Startup and Install. Preserves live settings by copying the existing app folder and avoids overwriting user runtime data during normal mirroring. |
| `SensorReadoutForm.Languages.cs` | Language loading and translation | Loads language files, detects Windows language, falls back to English, applies translated UI strings, tracks language changes, and resolves translated display text. | Language files live in `Langs` and use `key=value` entries. |
| `SensorReadoutForm.OemProviders.cs` | OEM/Plug-In bridge | Provides the sensor-collection hook for optional OEM hardware integrations. | This forwards to the plug-in loader instead of containing vendor-specific code. |
| `SensorReadoutForm.PlugIns.cs` | Plug-in loading | Discovers `plugin.json` files under `Plug-Ins`, reads optional manifest help links, respects per-machine enabled settings, loads enabled plug-in DLLs, catches plug-in failures, and converts SDK readings into normal `SensorRow` objects. | Installed plug-ins remain disabled until the user enables them. Manifest help links appear in Help only when that plug-in is enabled. |
| `SensorReadoutForm.PreferencesAndCommands.cs` | Commands and preferences integration | Opens Preferences, handles command/menu actions, applies updated settings, manages startup shortcut toggles, hidden-item commands, reading search, copy/rename/review actions, and report/menu entry points. | Connects main-window commands to settings and dialogs. |
| `SensorReadoutForm.ReadingTree.cs` | Reading tree UI | Builds and refreshes the category list and grouped reading tree, preserves expansion/selection state, handles context menu actions, exposes progress values for percentage readings, and updates visible rows. | Controls what users and screen readers navigate in the main reading area. |
| `SensorReadoutForm.ReportImport.cs` | Static report viewing and title text | Opens saved TXT/HTML reports, opens ZIP files that contain TXT/HTML reports, reads embedded report snapshots, falls back to legacy report parsing where possible, builds the main title bar text, switches the main tree into static report mode, and returns to live readings on request. | Lets support users inspect another machine's report through the normal Sensor Readout UI without manually extracting zipped reports first. |
| `SensorReadoutForm.ReportsAndLogging.cs` | Reports and logging | Writes TXT/HTML reports, formats report output, opens save dialogs, manages log file paths, writes diagnostics, and includes HTML escaping/report helpers. | Logging is optional and stored under `Logs` when enabled. |
| `SensorReadoutForm.SensorCollection.cs` | Sensor collection | Collects readings from LibreHardwareMonitor, Windows/WMI, network interfaces, logical drives, SMART/storage helpers, GPU registry data, Core Temp shared memory, fan controls, batteries, and plug-ins, then groups related rows into the app's category model. | This is where raw hardware/OS data becomes `SensorRow` objects; Wi-Fi radio details are delegated to the Wi-Fi partial. |
| `SensorReadoutForm.SettingsTransfer.cs` | Settings transfer | Exports and imports selective settings/profile packages, shows category checklists, resolves portable reading keys against the current machine, and imports hardware-bound items without active bindings. | Used by File > Export settings and profiles and File > Import settings and profiles. |
| `SensorReadoutForm.SettingsAndFiles.cs` | Settings and folders | Loads/saves per-computer JSON settings, migrates old top-level config/log files into `Config` and `Logs`, resolves app folders case-insensitively, and manages hidden reading persistence. | Keeps portable folder organization and backward compatibility together. |
| `SensorReadoutForm.StartupAndPrerequisites.cs` | Startup helpers and prerequisites | Checks/installs prerequisites, detects PawnIO, launches installer flow, offers optional Core Temp support, manages admin/elevation-related startup checks, and schedules first-refresh behavior. | Supports first-run and missing-driver guidance. |
| `SensorReadoutForm.Updates.cs` | Update checking and installer flow | Checks GitHub Releases, parses update metadata, shows update-available dialog, supports quiet update installs, formats release notes, downloads ZIP assets, and runs the self-update script. | The updater preserves user-modified language files by comparing bundled language hashes from the previous and incoming app packages. Used by Help > Check for updates and optional startup update checks. |
| `SensorReadoutForm.Usb.cs` | USB inventory and details | Collects USB devices, hubs, controllers, speeds, power where exposed, drive letters, safe-to-unplug status, USB network MAC/OUI data, storage hardware IDs, and detailed copyable fields. | Uses Windows device information plus the bundled `usb.ids` and OUI databases. |
| `SensorReadoutForm.Wifi.cs` | Wi-Fi radio details | Uses the Windows WLAN API to collect connected Wi-Fi details such as SSID, access point, signal percentage, RSSI, channel, frequency, radio type, link speeds, and security fields. | Rows are merged into the normal Network category and can be selected for tray or spoken hotkey output. |
| `UsbIdDatabase.cs` | USB vendor/product lookup | Loads the bundled `usb.ids` file and resolves USB VID/PID pairs into friendlier vendor and product names. | Used by the USB category and USB detail view. |
| `PlugInZipImporter.cs` | Plug-in ZIP import | Validates imported plug-in ZIP files, finds the manifest, checks the plug-in id, prompts before replacing an existing plug-in folder, and copies the plug-in into `Plug-Ins`. | Imported plug-ins stay disabled until the user enables them in Preferences. |

| Source file | Main responsibility | What it does | Notes |
| --- | --- | --- | --- |
| `src\PluginSdk\PluginSdk.cs` | Plug-in SDK | Defines `ISensorReadoutPlugin`, `IPluginContext`, `PluginInfo`, `MachineInfo`, and `SensorReading` for third-party plug-ins. | Built as `SensorReadout.PluginSdk.dll` and shipped beside the app. |
| `PlugIns\Framework\plugin.json` | Framework manifest | Describes the Framework plug-in and declares Help-menu support links for Framework BIOS and Drivers plus Framework Control. | Manifest links appear only when the plug-in is enabled. |
| `PlugIns\Framework\src\FrameworkPlugIn.cs` | Framework Laptop plug-in | Reads Framework Control's local API, and optional Framework EC helper tools, to expose Framework-specific temperature and fan rows. | Shipped disabled by default like every plug-in. |
| `PlugIns\HP\plugin.json` | HP manifest | Describes the HP plug-in and declares Help-menu support links for HP Support Assistant and OMEN Gaming Hub. | Manifest links appear only when the plug-in is enabled. |
| `PlugIns\HP\src\HPPlugIn.cs` | HP hardware plug-in | Checks HP/OMEN/Victus WMI interfaces for optional OEM hardware diagnostics. | Experimental, read-only, and disabled by default. |
| `PlugIns\DellLatitude\plugin.json` | Dell manifest | Describes the Dell Latitude plug-in and declares a Help-menu support link for Dell Command \| Monitor. | Manifest links appear only when the plug-in is enabled. |
| `PlugIns\DellLatitude\src\DellLatitudePlugIn.cs` | Dell Latitude plug-in | Checks Dell Command \| Monitor WMI classes for additional Dell temperature and fan readings when the user enables the plug-in. | Experimental, read-only, and disabled by default. |
| `PlugIns\AsusRog\plugin.json` | Asus manifest | Describes the Asus ROG/TUF plug-in and declares a Help-menu support link for G-Helper. | Manifest links appear only when the plug-in is enabled. |
| `PlugIns\AsusRog\src\AsusRogPlugIn.cs` | Asus ROG/TUF plug-in | Checks Asus ATKACPI/WMI interfaces and attempts supported Asus fan control through experimental ACPI integration. | Experimental, disabled by default, and includes its own third-party notice/GPL text. |
