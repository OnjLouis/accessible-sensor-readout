# Sensor Readout Source File Map

This table explains the current C# source split used by Sensor Readout. It is intended for contributors and anyone who wants to understand where a feature lives before reading the code. Keep it updated whenever files are added, removed, renamed, or given a materially different responsibility.

| Source file | Main responsibility | What it does | Notes |
| --- | --- | --- | --- |
| `AssemblyInfo.cs` | Assembly metadata | Sets the application assembly title and version numbers compiled into the executable. | Small metadata file; version should match the public release. |
| `Models.cs` | Shared data models | Defines the plain data objects used across the app: settings, sensor rows, tray selections, fan labels, spoken hotkey profiles, fan profiles, alarms, plug-in preferences, plug-in help links, update info, update confirmation preferences, and language choices. | Keeps cross-feature data contracts away from UI code. |
| `MacVendorDatabase.cs` | MAC/OUI vendor lookup | Loads the bundled OUI vendor database from `Data` and resolves MAC address or OUI-style prefixes to vendor names. | Used by the Network category and USB network/storage details. |
| `NativeInterop.cs` | Windows and screen-reader interop | Contains P/Invoke declarations and small wrappers for Windows APIs, global hotkeys, shutdown handling, notification-area tooltip updates, memory-mapped Core Temp data, Tolk screen-reader speech, and related native structures. | This is the boundary between managed C# and native DLL/Windows calls. |
| `Program.cs` | Application startup and command line | Parses command-line options, handles single-instance and close behavior, report-only modes, diagnostics-only mode, build-time self-test mode, logging override, TLS setup, crash logging/restart handoff, and starts the WinForms message loop. | Entry point for `Sensor Readout.exe`; close-request signalling is reset after close handling so later launches do not inherit a stale close request. |
| `PreferencesForm.cs` | Preferences window shell | Defines Preferences fields, constructor layout, tab creation, primary control wiring, and live-setting event hooks. | Partial class root for the Preferences dialog. Feature-specific behavior is split into the focused `PreferencesForm.*.cs` files below. |
| `PreferencesForm.Core.cs` | Preferences shared behavior | Handles Preferences language refresh, dialog keyboard routing, fixed option text, live saving, desktop shortcut preference application, and shared combo/hotkey helpers. | Shared by every Preferences tab. |
| `PreferencesForm.Panels.cs` | Preferences panel builders | Builds the tray selection, spoken hotkey, fan profile, sound picker, alarms, plug-ins, hidden items, and language editor panels. Also handles plug-in details/import and language-editor file operations. | Construction-only code is kept away from list editing logic where practical. |
| `PreferencesForm.TraySearch.cs` | Tray selection and Preferences list search | Manages notification-area reading selection, ordering, rename, status text, list search, live row refresh, and tray choice population. | Used by the General tab and reusable F3-style list search. |
| `PreferencesForm.SpokenHotKeys.cs` | Spoken hotkey preferences | Creates/removes/imports spoken hotkey profiles, resolves imported reading keys, edits selected readings, spoken labels, ordering, and preview text. | Keeps spoken-hotkey setup separate from tray status setup. |
| `PreferencesForm.FanProfiles.cs` | Fan profile preferences | Creates/removes fan profiles, edits profile headers/actions, applies a profile from Preferences, filters fan controls, and formats fan-control display names. | Fan runtime behavior remains in `SensorReadoutForm.FanControls.cs` and `SensorReadoutForm.FanCurves.cs`. |
| `PreferencesForm.Alarms.cs` | Alarm preferences | Builds alarm reading choices, creates/removes/loads/saves alarms, handles threshold units, threshold conversion, cooldowns, and alarm list text. | Alarm runtime evaluation remains in `SensorReadoutForm.Alarms.cs`. |
| `PreferencesForm.Helpers.cs` | Preferences helper models | Returns current Preferences state, clones mutable settings, handles hidden keys and speech labels, defines list choice comparers, and contains small nested list item/helper types. | Kept separate so the larger workflow files stay focused. |
| `SensorReadoutForm.cs` | Main window shell | Defines the main form fields and constructor, creates the primary UI layout, menu structure, category list, tree view, status/progress area, timers, and high-level event wiring. | The central partial class that the feature-specific files extend. |
| `SensorReadoutForm.Alarms.cs` | Alarms and sounds runtime | Evaluates alarm rules against current readings, triggers screen-reader speech and WAV playback, flashes the tray icon, plays startup/shutdown sounds, and loads sound file names from the `Sounds` folder. | Uses WAV playback through .NET `SoundPlayer`. |
| `SensorReadoutForm.Audio.cs` | Audio device inventory | Reads Windows sound devices and audio endpoint data, including vendor, status, playback/recording direction, and default channel/sample-rate/bit-depth format where exposed. | Used by the Audio category and reports. |
| `SensorReadoutForm.Battery.cs` | Battery readings | Reads battery charge, status, capacity, health, cycle count, voltage, and power-rate information where Windows exposes it. | Used by laptops and other systems with batteries. |
| `SensorReadoutForm.Dispose.cs` | Cleanup | Disposes timers, tray icon, hardware computer objects, hidden windows, and other resources when the app exits. | Keeps shutdown cleanup separate from feature logic. |
| `SensorReadoutForm.Devices.cs` | Windows device inventory | Collects Device Manager-style PnP/driver/resource rows for the Devices category and duplicates problem devices into a top Non-working devices group. | Uses Win32_PnPEntity, Win32_PnPSignedDriver, Win32_PNPAllocatedResource, and selected Enum registry fields; keeps noisy IDs in Details and decodes common Device Manager problem codes. |
| `SensorReadoutForm.Display.cs` | Display inventory | Reads graphics adapters and monitor identity data, including adapter memory, current resolution, driver information, monitor vendor, product code, serial, and manufacture date where exposed. | Used by the Display category and reports. |
| `SensorReadoutForm.Diagnostics.cs` | One-click diagnostics | Runs Help-menu diagnostics, temporarily enables debug-level logging in memory, creates TXT/HTML reports, exercises writable fan controls, restores fan/manual/curve state, writes a summary, zips the diagnostic bundle with the computer name in the file name, removes the temporary staging folder, and opens the Reports folder. | Designed for support cases where users need to send one ZIP instead of several separate files. |
| `SensorReadoutForm.FanControls.cs` | Fan controls dialog and refresh pipeline | Builds and manages the Fan Controls dialog, including fan selection, friendly labels, manual percentages, automatic/default control, show-stopped behavior, and visible fan command buttons. Also coordinates refresh collection, separating fast live rows from cached slow inventory rows. | Fan control depends on LibreHardwareMonitor/PawnIO and writable controls exposed by hardware. Slow refresh rows include SMART, USB, audio, display, device inventory, CPU inventory, and overview data so 1-second refreshes do not repeatedly query heavier hardware/WMI paths. |
| `SensorReadoutForm.FanCurves.cs` | Fan curves | Builds and manages the Fan Curves dialog, stores curve definitions, and applies enabled curves by mapping selected temperature readings to writable fan control percentages. | Curve support still depends on hardware exposing writable fan controls. |
| `SensorReadoutForm.Formatting.cs` | Display formatting | Formats numbers, temperatures, bytes, rates, percentages, uptime, fan display text, speech labels, selected meter values, and shared rules for which rows can be used as quick readouts. | Keeps text/value presentation consistent across UI, tray, reports, hotkeys, and alarms, including GPU memory and GPU engine rows that are safe to expose as selectable readouts. |
| `SensorReadoutForm.HotkeysTrayHelp.cs` | Hotkeys, tray, help, and screen-reader speech | Registers and handles global hotkeys, builds the Hotkeys menu, handles show/hide behavior, spoken tray status, custom spoken hotkey profiles, double-press clipboard copy, configurable tray tooltip overflow behavior, cached tray tooltip/icon updates, Help/About/Donate/Contact links, version history, dynamic enabled plug-in help links, and F1 manual opening. | Main accessibility and notification-area interaction layer. Tray and hotkey row resolution uses the current row index first so frequent refreshes do not repeatedly scan the full row list. |
| `SensorReadoutForm.Install.cs` | Install-to-PC and uninstall flow | Detects whether the app is running from the local install folder, copies the current app folder into the normal Windows programs folder for the current user, manages desktop and startup shortcuts, closes the old copy, relaunches from the installed location, and removes installed app files during uninstall with an explicit option to also delete Config/Logs/Reports. | Used by Preferences > Startup and Install. Preserves live settings by copying the existing app folder and avoids overwriting user runtime data during normal mirroring. |
| `SensorReadoutForm.Languages.cs` | Language loading and translation | Loads language files, detects Windows language, falls back to English, applies translated UI strings, tracks language changes, and resolves translated display text. | Language files live in `Langs` and use `key=value` entries. |
| `SensorReadoutForm.OemProviders.cs` | OEM/Plug-In bridge | Provides the sensor-collection hook for optional OEM hardware integrations. | This forwards to the plug-in loader instead of containing vendor-specific code. |
| `SensorReadoutForm.PlugIns.cs` | Plug-in loading | Discovers `plugin.json` files under `Plug-Ins`, reads optional manifest help links, respects per-machine enabled settings, loads enabled plug-in DLLs, catches plug-in failures, and converts SDK readings into normal `SensorRow` objects. | Installed plug-ins remain disabled until the user enables them. Manifest help links appear in Help only when that plug-in is enabled. |
| `SensorReadoutForm.PortableExport.cs` | Portable copy export | Creates a ZIP archive of the current app folder for use on another PC, including settings, Plug-Ins, language files, sounds, and other app files, with optional Reports and Logs. | Used by File > Export portable copy. Writes to a temporary ZIP first so the chosen destination is not accidentally included in the archive, and skips update temp/backup folders. |
| `SensorReadoutForm.Printers.cs` | Printer overview | Reads Windows printer status, drivers, ports, sharing, print configuration, queued jobs, and driver-exposed ink/toner percentages where available. | Adds printer summary rows to Performance/Overview and per-printer rows to the Performance > Printers subtree. Offline state, queued jobs, and usable supply percentages can be selected for tray, spoken hotkeys, or alarms. |
| `SensorReadoutForm.PreferencesAndCommands.cs` | Commands and preferences integration | Opens Preferences, handles command/menu actions, applies updated settings, manages startup shortcut toggles, hidden-item commands, reading search, copy/copy-value/rename/review actions, quick notification-area/spoken-hotkey assignment from the selected reading, and report/menu entry points. | Connects main-window commands to settings and dialogs. |
| `SensorReadoutForm.ReadingTree.cs` | Reading tree UI | Builds and refreshes the category list and grouped reading tree, preserves expansion/selection state, handles context menu actions, exposes progress values for percentage readings, announces when selected rows have Details, and updates visible rows. | Controls what users and screen readers navigate in the main reading area without adding accessibility hints to copied row text, including the Performance subgroups for system, graphics, printers, and storage. |
| `SensorReadoutForm.ReportImport.cs` | Static report viewing and title text | Opens saved TXT/HTML reports, opens ZIP files that contain TXT/HTML reports, reads embedded report snapshots, preserves imported report title/machine context, falls back to legacy report parsing where possible, builds the main title bar text, switches the main tree into static report mode, and returns to live readings on request. | Lets support users inspect another machine's report through the normal Sensor Readout UI without manually extracting zipped reports first. |
| `SensorReadoutForm.ReportsAndLogging.cs` | Reports and logging | Writes TXT/HTML reports, formats report output, opens save dialogs, manages log file paths, writes diagnostics, and includes HTML escaping/report helpers. | TXT reports stay human-readable. HTML reports include the hidden structured snapshot used for exact report reopening. Logging is optional and stored under `Logs` when enabled. |
| `SensorReadoutForm.SensorCollection.cs` | Sensor collection | Collects readings from LibreHardwareMonitor, Windows/WMI, network interfaces, logical drives, SMART/storage helpers, direct NVMe SMART health-log fallbacks, GPU registry/performance-counter data, Core Temp shared memory, fan controls, batteries, printer overview rows, and plug-ins, then groups related rows into the app's category model. | This is where raw hardware/OS data becomes `SensorRow` objects, including richer CPU/cache, baseboard/enclosure, expansion slot, memory array/module, Windows Storage physical-disk fields, GPU memory usage rows, storage partition/volume Details, and network adapter Details; Wi-Fi radio details are delegated to the Wi-Fi partial and printer rows to the Printers partial. |
| `SensorReadoutForm.SelfTest.cs` | Internal build-time self-tests | Runs the non-interactive `--self-test` workflow used by `Build.ps1 -SelfTest`, exercising settings persistence, collection, tree navigation, expansion state, tray text, quick tray/spoken hotkey assignments, alarms, fan curves, reports, ZIP report selection, diagnostics ZIP creation, and language/manual presence. | Intended for release and development checks on a disposable app copy; not a user-facing diagnostic feature. |
| `SensorReadoutForm.SettingsTransfer.cs` | Settings transfer | Exports and imports selective settings/profile packages, shows category checklists, resolves portable reading keys against the current machine, and imports hardware-bound items without active bindings. | Used by File > Export settings and profiles and File > Import settings and profiles. |
| `SensorReadoutForm.SettingsAndFiles.cs` | Settings and folders | Loads/saves per-computer JSON settings, migrates old top-level config/log files into `Config` and `Logs`, resolves app folders case-insensitively, and manages hidden reading persistence. | Keeps portable folder organization and backward compatibility together. |
| `SensorReadoutForm.StartupAndPrerequisites.cs` | Startup helpers and prerequisites | Checks/installs prerequisites, detects PawnIO, launches installer flow, offers optional Core Temp support, manages admin/elevation-related startup checks, creates/removes the Windows logon scheduled task used for Run at startup, cleans up legacy startup shortcuts/Run-key entries, and schedules first-refresh behavior. | Supports first-run and missing-driver guidance. The startup task runs with highest privileges so installed copies can start at logon despite the app's administrator manifest. |
| `SensorReadoutForm.Updates.cs` | Update checking and installer flow | Checks GitHub Releases, parses update metadata, shows update-available dialog, supports quiet update installs, handles the optional close/install/reopen confirmation, formats release notes, downloads ZIP assets, and runs the self-update script. | The updater preserves user-modified language files by comparing bundled language hashes from the previous and incoming app packages, and never copies Config, Logs, or Reports from an update ZIP over an existing installation. Used by Help > Check for updates and optional startup update checks. |
| `SensorReadoutForm.Usb.cs` | USB inventory and details | Collects USB devices, hubs, controllers, speeds, power where exposed, drive letters, safe-to-unplug status, USB network MAC/OUI data, storage hardware IDs, and detailed copyable fields. | Uses Windows device information plus the bundled `usb.ids` and OUI databases; also hosts the shared Details dialog as a grouped tree so raw WMI/registry fields do not interrupt practical fields. |
| `SensorReadoutForm.Wifi.cs` | Wi-Fi radio details | Uses the Windows WLAN API to collect connected Wi-Fi details such as SSID, access point, signal percentage, RSSI, channel, frequency, radio type, link speeds, and security fields. | Rows are merged into the normal Network category and can be selected for tray or spoken hotkey output. |
| `UsbIdDatabase.cs` | USB vendor/product lookup | Loads the bundled `usb.ids` file and resolves USB VID/PID pairs into friendlier vendor and product names. | Used by the USB category and USB detail view. |
| `PlugInZipImporter.cs` | Plug-in ZIP import | Validates imported plug-in ZIP files, finds the manifest, checks the plug-in id, prompts before replacing an existing plug-in folder, and copies the plug-in into `Plug-Ins`. | Imported plug-ins stay disabled until the user enables them in Preferences. |

| Source file | Main responsibility | What it does | Notes |
| --- | --- | --- | --- |
| `src\PluginSdk\PluginSdk.cs` | Plug-in SDK | Defines `ISensorReadoutPlugin`, `IPluginContext`, `PluginInfo`, `MachineInfo`, and `SensorReading` for third-party plug-ins. | Built as `SensorReadout.PluginSdk.dll` and shipped beside the app. |
| `PlugIns\Framework\plugin.json` | Framework manifest | Describes the Framework plug-in and declares Help-menu support links for Framework BIOS and Drivers plus Framework Control. | Manifest links appear only when the plug-in is enabled. |
| `PlugIns\Framework\src\FrameworkPlugIn.cs` | Framework Laptop plug-in | Reads Framework Control's local API, and optional Framework EC helper tools, to expose Framework-specific temperature and fan rows. | Shipped disabled by default like every plug-in. |
| `PlugIns\HP\plugin.json` | HP manifest | Describes the HP plug-in and declares Help-menu support links for HP Support Assistant and OMEN Gaming Hub. | Manifest links appear only when the plug-in is enabled. |
| `PlugIns\HP\src\HPPlugIn.cs` | HP hardware plug-in | Checks HP/OMEN/Victus WMI interfaces for optional OEM hardware diagnostics. | Experimental, read-only, and disabled by default. |
| `PlugIns\DellLatitude\plugin.json` | Dell manifest | Describes the Dell Latitude plug-in and declares a Help-menu support link for Dell Command \| Monitor. | Manifest links appear only when the plug-in is enabled. |
| `PlugIns\DellLatitude\src\DellLatitudePlugIn.cs` | Dell Latitude plug-in | Checks Dell Command \| Monitor WMI classes for additional Dell temperature and fan readings when the user enables the plug-in. | Experimental, read-only, and disabled by default. |
| `PlugIns\AsusRog\plugin.json` | Asus manifest | Describes the Asus Laptop Support plug-in and declares a Help-menu support link for G-Helper. | Manifest links appear only when the plug-in is enabled. |
| `PlugIns\AsusRog\src\AsusRogPlugIn.cs` | Asus laptop plug-in | Checks Asus ATKACPI first, falls back to ASUS WMI read probes, and attempts supported Asus fan control through experimental ATKACPI integration. | Experimental, disabled by default, read-only when using ASUS WMI, and includes its own third-party notice/GPL text. |
