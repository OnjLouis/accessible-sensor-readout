# Sensor Readout Source File Map

This table explains the C# source split used by Sensor Readout 1.5.0. It is intended for contributors and anyone who wants to understand where a feature lives before reading the code.

| Source file | Main responsibility | What it does | Notes |
| --- | --- | --- | --- |
| `AssemblyInfo.cs` | Assembly metadata | Sets the application assembly title and version numbers compiled into the executable. | Small metadata file; version should match the public release. |
| `Models.cs` | Shared data models | Defines the plain data objects used across the app: settings, sensor rows, tray selections, fan labels, spoken hotkey profiles, alarms, update info, and language choices. | Keeps cross-feature data contracts away from UI code. |
| `MacVendorDatabase.cs` | MAC vendor lookup | Loads the bundled OUI vendor database from `Data` and resolves network adapter MAC address prefixes to vendor names. | Used by the Network category. |
| `NativeInterop.cs` | Windows and screen-reader interop | Contains P/Invoke declarations and small wrappers for Windows APIs, global hotkeys, shutdown handling, memory-mapped Core Temp data, Tolk screen-reader speech, and related native structures. | This is the boundary between managed C# and native DLL/Windows calls. |
| `Program.cs` | Application startup and command line | Parses command-line options, handles single-instance and close behavior, report-only modes, logging override, TLS setup, and starts the WinForms message loop. | Entry point for `Sensor Readout.exe`. |
| `PreferencesForm.cs` | Preferences window | Builds and manages Preferences: General, Startup, Hotkeys, Alarms, Hidden items, and Language editor tabs. Handles immediate saving, accessible labels, list ordering, alarm editing, sound preview, and keyboard navigation inside Preferences. | Largest file because the preferences dialog has many controls and workflows. |
| `SensorReadoutForm.cs` | Main window shell | Defines the main form fields and constructor, creates the primary UI layout, menu structure, category list, tree view, status/progress area, timers, and high-level event wiring. | The central partial class that the feature-specific files extend. |
| `SensorReadoutForm.Alarms.cs` | Alarms and sounds runtime | Evaluates alarm rules against current readings, triggers screen-reader speech and WAV playback, flashes the tray icon, plays startup/shutdown sounds, and loads sound file names from the `Sounds` folder. | Uses WAV playback through .NET `SoundPlayer`. |
| `SensorReadoutForm.Dispose.cs` | Cleanup | Disposes timers, tray icon, hardware computer objects, hidden windows, and other resources when the app exits. | Keeps shutdown cleanup separate from feature logic. |
| `SensorReadoutForm.FanControls.cs` | Fan controls dialog | Builds and manages the Fan Controls dialog, including fan selection, friendly labels, manual percentages, automatic/default control, show-stopped behavior, and visible fan command buttons. | Fan control depends on LibreHardwareMonitor/PawnIO and writable controls exposed by hardware. |
| `SensorReadoutForm.FanCurves.cs` | Fan curves | Builds and manages the Fan Curves dialog, stores curve definitions, and applies enabled curves by mapping selected temperature readings to writable fan control percentages. | Curve support still depends on hardware exposing writable fan controls. |
| `SensorReadoutForm.Formatting.cs` | Display formatting | Formats numbers, temperatures, bytes, rates, percentages, uptime, fan display text, speech labels, and selected meter values. | Keeps text/value presentation consistent across UI, tray, reports, hotkeys, and alarms. |
| `SensorReadoutForm.HotkeysTrayHelp.cs` | Hotkeys, tray, help, and screen-reader speech | Registers and handles global hotkeys, show/hide behavior, spoken tray status, custom spoken hotkey profiles, double-press clipboard copy, tray icon behavior, Help/About/Donate/Contact links, and F1 manual opening. | Main accessibility and notification-area interaction layer. |
| `SensorReadoutForm.Languages.cs` | Language loading and translation | Loads language files, detects Windows language, falls back to English, applies translated UI strings, tracks language changes, and resolves translated display text. | Language files live in `Langs` and use `key=value` entries. |
| `SensorReadoutForm.PreferencesAndCommands.cs` | Commands and preferences integration | Opens Preferences, handles command/menu actions, applies updated settings, manages startup shortcut toggles, hidden-item commands, copy/rename actions, and report/menu entry points. | Connects main-window commands to settings and dialogs. |
| `SensorReadoutForm.ReadingTree.cs` | Reading tree UI | Builds and refreshes the category list and reading tree, preserves expansion/selection state, handles context menu actions, exposes progress values for percentage readings, and updates visible rows. | Controls what users and screen readers navigate in the main reading area. |
| `SensorReadoutForm.ReportsAndLogging.cs` | Reports and logging | Writes TXT/HTML reports, formats report output, opens save dialogs, manages log file paths, writes diagnostics, and includes HTML escaping/report helpers. | Logging is optional and stored under `Logs` when enabled. |
| `SensorReadoutForm.SensorCollection.cs` | Sensor collection | Collects readings from LibreHardwareMonitor, Windows/WMI, network interfaces, logical drives, SMART/storage helpers, GPU registry data, Core Temp shared memory, and fan control sensors. | This is where raw hardware/OS data becomes `SensorRow` objects. |
| `SensorReadoutForm.SettingsAndFiles.cs` | Settings and folders | Loads/saves per-computer JSON settings, migrates old top-level config/log files into `Config` and `Logs`, resolves app folders case-insensitively, and manages hidden reading persistence. | Keeps portable folder organization and backward compatibility together. |
| `SensorReadoutForm.StartupAndPrerequisites.cs` | Startup helpers and prerequisites | Checks/installs prerequisites, detects PawnIO, launches installer flow, manages admin/elevation-related startup checks, and schedules first-refresh behavior. | Supports first-run and missing-driver guidance. |
| `SensorReadoutForm.Updates.cs` | Update checking and installer flow | Checks GitHub Releases, parses update metadata, shows update-available dialog, formats release notes, downloads ZIP assets, and runs the self-update script. | Used by Help > Check for updates and optional startup update checks. |
| `SensorReadoutForm.Usb.cs` | USB inventory and details | Collects USB devices, hubs, controllers, speeds, power where exposed, drive letters, safe-to-unplug status, and detailed copyable fields. | Uses Windows device information plus the bundled `usb.ids` database. |
| `UsbIdDatabase.cs` | USB vendor/product lookup | Loads the bundled `usb.ids` file and resolves USB VID/PID pairs into friendlier vendor and product names. | Used by the USB category and USB detail view. |
