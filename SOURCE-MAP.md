# Sensor Readout Source File Map

This table explains the current C# source split used by Sensor Readout. It is intended for contributors and anyone who wants to understand where a feature lives before reading the code. Keep it updated whenever files are added, removed, renamed, or given a materially different responsibility.

| Source file | Main responsibility | What it does | Notes |
| --- | --- | --- | --- |
| `AssemblyInfo.cs` | Assembly metadata | Sets the main executable title, product metadata, and assembly/file/product version numbers compiled into the executable. | `Build.ps1` verifies this against `SensorReadoutForm.AppVersion` and the app manifest. |
| `Build.ps1` | Local build and self-test | Compiles the EXE, plug-in SDK, and bundled plug-ins; generates shared assembly metadata for DLLs; validates AppVersion, AssemblyInfo, manifest, EXE, SDK DLL, and plug-in DLL version parity; updates bundled language hashes; clears preserved portable folders; and can run the internal self-test. | The build fails if any Sensor Readout binary reports a stale file or product version. |
| `Release.ps1` | Release gate and packaging | Runs version, changelog, GitHub activity, language, manual-health, build, report, diagnostics, ZIP-shape, upgrade, mirror, and post-publish updater checks before packaging or release. | Manual-health checks now fail on stale top-of-manual versions, mojibake markers, abnormal localized manual size, and extremely long generated lines. |
| `Update-Data.ps1` | Bundled vendor database updater | Downloads and validates the latest `usb.ids` and IEEE `oui.csv` snapshots, updates `Data`, and mirrors them to `portable\Data`. | Called by the release gate so USB and MAC vendor names stay current without making ordinary compiles depend on the network. |
| `Models.cs` | Shared data models | Defines the plain data objects used across the app: settings, sensor rows, tray selections, fan labels, spoken hotkey profiles, fan profiles, alarms, plug-in preferences, plug-in help links, reading-history selections, update info, update confirmation preferences, and language choices. | Keeps cross-feature data contracts away from UI code. |
| `MacVendorDatabase.cs` | MAC/OUI vendor lookup | Loads the bundled OUI vendor database from `Data` and resolves MAC address or OUI-style prefixes to vendor names. | Used by the Network category and USB network/storage details. |
| `NativeInterop.cs` | Windows and screen-reader interop | Contains P/Invoke declarations and small wrappers for Windows APIs, global hotkeys, shutdown handling, notification-area tooltip updates, memory-mapped Core Temp data, Tolk screen-reader speech, and related native structures. | This is the boundary between managed C# and native DLL/Windows calls. |
| `Program.cs` | Application startup and command line | Parses command-line options, handles single-instance and close behavior, normal/anonymized report-only modes, report comparison mode, diagnostics-only mode, build-time self-test mode, logging override, TLS setup, crash logging/restart handoff, and starts the WinForms message loop. | Entry point for `Sensor Readout.exe`; close-request signalling is reset after close handling so later launches do not inherit a stale close request. |
| `PreferencesForm.cs` | Preferences window shell | Defines Preferences fields, constructor layout, tab creation, primary control wiring, and live-setting event hooks. | Partial class root for the Preferences dialog. Feature-specific behavior is split into the focused `PreferencesForm.*.cs` files below. |
| `PreferencesForm.Core.cs` | Preferences shared behavior | Handles Preferences language refresh, dialog keyboard routing, fixed option text, live saving, desktop shortcut preference application, and shared combo/hotkey helpers. | Shared by every Preferences tab. |
| `PreferencesForm.Panels.cs` | Preferences panel builders | Builds the tray selection, spoken hotkey, fan profile, sound picker, alarms, plug-ins, hidden items, and language editor panels. Also handles plug-in details/import, language-editor search, language-editor cursor restore, and language-editor file operations. | Construction-only code is kept away from list editing logic where practical. |
| `PreferencesForm.TraySearch.cs` | Tray selection and Preferences list search | Manages notification-area reading selection, ordering, rename, status text, list search, live row refresh, and tray choice population. | Used by the General tab and reusable F3-style list search. |
| `PreferencesForm.SpokenHotKeys.cs` | Spoken hotkey preferences | Creates/removes/imports spoken hotkey profiles, resolves imported reading keys, edits selected readings, spoken labels, ordering, preset profiles, removal confirmation, and preview text. | Keeps spoken-hotkey setup separate from tray status setup; profile lists are sorted by assigned key for predictable navigation. |
| `PreferencesForm.FanProfiles.cs` | Fan profile preferences | Creates/removes fan profiles, edits profile headers/actions, applies a profile from Preferences, filters fan controls, and formats fan-control display names. | Fan runtime behavior remains in `SensorReadoutForm.FanControls.cs` and `SensorReadoutForm.FanCurves.cs`. |
| `PreferencesForm.Alarms.cs` | Alarm preferences | Builds alarm reading choices, creates/removes/loads/saves alarms, handles threshold units, threshold conversion, cooldowns, and alarm list text. | Alarm runtime evaluation remains in `SensorReadoutForm.Alarms.cs`. |
| `PreferencesForm.Helpers.cs` | Preferences helper models | Returns current Preferences state, clones mutable settings, handles hidden keys and speech labels, defines list choice comparers, and contains small nested list item/helper types. | Kept separate so the larger workflow files stay focused. |
| `SensorReadoutForm.cs` | Main window shell | Defines the main form fields and constructor, creates the primary UI layout, menu structure, category list, tree view, status/progress area, timers, and high-level event wiring. | The central partial class that the feature-specific files extend. |
| `SensorReadoutForm.Alarms.cs` | Alarms and sounds runtime | Evaluates alarm rules against current readings, triggers screen-reader speech and WAV playback, flashes the tray icon, plays startup/shutdown sounds, and loads sound file names from the `Sounds` folder. | Uses WAV playback through .NET `SoundPlayer`. |
| `SensorReadoutForm.Audio.cs` | Audio device inventory | Reads Windows sound devices and audio endpoint data, including vendor, status, playback/recording direction, and default channel/sample-rate/bit-depth format where exposed. | Used by the Audio category and reports. |
| `SensorReadoutForm.Battery.cs` | Battery readings | Reads battery charge, status, capacity, health, cycle count, voltage, and power-rate information where Windows exposes it. | Used by laptops and other systems with batteries. |
| `SensorReadoutForm.Dispose.cs` | Cleanup | Disposes timers, tray icon, hardware computer objects, hidden windows, and other resources when the app exits. | Keeps shutdown cleanup separate from feature logic. |
| `SensorReadoutForm.Devices.cs` | Windows device inventory | Collects Device Manager-style PnP/driver/resource rows for the Devices category and duplicates problem devices into a top Non-working devices group. | Uses Win32_PnPEntity, Win32_PnPSignedDriver, Win32_PNPAllocatedResource, and selected Enum registry fields; keeps noisy IDs in Details and decodes common Device Manager problem codes. |
| `SensorReadoutForm.Display.cs` | Display inventory | Reads graphics adapters and monitor identity data, including adapter memory, current resolution, driver information, monitor vendor, product code, serial, and manufacture date where exposed. | Used by the Display category and reports. |
| `SensorReadoutForm.Diagnostics.cs` | One-click diagnostics | Runs Help-menu diagnostics, temporarily enables debug-level logging in memory, creates TXT/HTML reports, records plug-in enabled/disabled state, exercises writable fan controls, restores fan/manual/curve state, writes a summary, zips the diagnostic bundle with the computer name in the file name, removes the temporary staging folder, and opens the Reports folder. | Designed for support cases where users need to send one ZIP instead of several separate files, including enough plug-in state to tell whether matching laptop support was enabled during the run. |
| `SensorReadoutForm.FanControls.cs` | Fan controls dialog and refresh pipeline | Builds and manages the Fan Controls dialog, including fan selection, friendly labels, manual percentages, automatic/default control, show-stopped behavior, and visible fan command buttons. Also coordinates refresh collection, separating fast live rows from cached slow inventory rows and timing out slow LibreHardwareMonitor passes so Windows-backed rows can still appear. | Fan control depends on LibreHardwareMonitor/PawnIO and writable controls exposed by hardware. Slow refresh rows include SMART, USB, audio, display, device inventory, CPU inventory, and overview data so 1-second refreshes do not repeatedly query heavier hardware/WMI paths. |
| `SensorReadoutForm.FanCurves.cs` | Fan curves | Builds and manages the Fan Curves dialog, stores curve definitions, and applies enabled curves by mapping selected temperature readings to writable fan control percentages. | Curve support still depends on hardware exposing writable fan controls. |
| `SensorReadoutForm.Formatting.cs` | Display formatting | Formats numbers, temperatures, bytes, rates, percentages, uptime, fan display text, speech labels, selected meter values, and shared rules for which rows can be used as quick readouts. | Keeps text/value presentation consistent across UI, tray, reports, hotkeys, and alarms, including GPU memory and GPU engine rows that are safe to expose as selectable readouts. |
| `SensorReadoutForm.HotkeysTrayHelp.cs` | Hotkeys, tray, help, and screen-reader speech | Registers and handles global hotkeys, builds the Hotkeys menu, handles show/hide behavior, spoken tray status, custom spoken hotkey profiles, report-mode hotkey wording, double-press clipboard copy, configurable tray tooltip overflow behavior, cached tray tooltip/icon updates, Help/About/Donate/Contact links, version history, dynamic enabled plug-in help links, and F1 manual opening. | Main accessibility and notification-area interaction layer. Tray and hotkey row resolution uses the current row index first so frequent refreshes do not repeatedly scan the full row list. |
| `SensorReadoutForm.Install.cs` | Install-to-PC and uninstall flow | Detects whether the app is running from the local install folder, copies the current app folder into the normal Windows programs folder for the current user, manages desktop and startup shortcuts, closes the old copy, relaunches from the installed location, and removes installed app files during uninstall with an explicit option to also delete Config/Logs/Reports. | Used by Preferences > Startup and Install. Preserves live settings by copying the existing app folder and avoids overwriting user runtime data during normal mirroring. |
| `SensorReadoutForm.Languages.cs` | Language loading and translation | Loads language files, detects Windows language, falls back to English, applies translated UI strings, tracks language changes, and resolves translated display text. | Language files live in `Langs` and use `key=value` entries. |
| `SensorReadoutForm.OemProviders.cs` | OEM/Plug-In bridge | Provides the sensor-collection hook for optional OEM hardware integrations. | This forwards to the plug-in loader instead of containing vendor-specific code. |
| `SensorReadoutForm.PlugIns.cs` | Plug-in loading | Discovers `plugin.json` files at the `Plug-Ins` root and one folder below it, reads optional manifest help links, respects per-machine enabled settings, loads enabled plug-in DLLs, logs disabled/loaded/failed state and returned row counts, catches plug-in failures, and converts SDK readings into normal `SensorRow` objects. | Installed plug-ins remain disabled until the user enables them. Duplicate plug-in ids and nested stale manifests are ignored defensively. Manifest help links appear in Help only when that plug-in is enabled. The shipped app config allows plug-ins to load from extracted downloaded ZIP builds that Windows marks as internet-origin files. |
| `SensorReadoutForm.PortableExport.cs` | Portable copy export | Creates a ZIP archive of the current app folder for use on another PC, including settings, Plug-Ins, language files, sounds, and other app files, with optional Reports and Logs. | Used by File > Export portable copy. Writes to a temporary ZIP first so the chosen destination is not accidentally included in the archive, and skips update temp/backup folders. |
| `SensorReadoutForm.Printers.cs` | Printer overview | Reads Windows printer status, drivers, ports, sharing, print configuration, queued jobs, and driver-exposed ink/toner percentages where available. | Adds printer summary rows to Performance/Overview and per-printer rows to the Performance > Printers subtree. Offline state, queued jobs, and usable supply percentages can be selected for tray, spoken hotkeys, or alarms. |
| `SensorReadoutForm.PreferencesAndCommands.cs` | Commands and preferences integration | Opens Preferences, handles command/menu actions, applies updated settings, manages startup shortcut toggles, hidden-item commands, reading search, copy/copy-value/rename/review actions, quick notification-area/spoken-hotkey assignment from the selected reading, and report/menu entry points. | Connects main-window commands to settings and dialogs. The quick assignment dialog formats notification-area status with its speak-tray hotkey and selected-reading count, matching spoken hotkey profile entries. |
| `SensorReadoutForm.ReadingTree.cs` | Reading tree UI | Builds and refreshes the category list and grouped reading tree, preserves expansion/selection state, handles context menu actions, exposes progress values for percentage readings, announces when selected rows have Details, and updates visible rows. | Controls what users and screen readers navigate in the main reading area without adding accessibility hints to copied row text, including the Performance subgroups for system, graphics, printers, and storage; static report placeholders avoid live-refresh wording. Rebuild decisions include the selected category so visually identical empty states do not leave the previous tree in place. |
| `SensorReadoutForm.ReportImport.cs` | Static report viewing and title text | Opens saved TXT/HTML reports, opens ZIP files that contain TXT/HTML reports, reads embedded report snapshots, preserves imported report title/machine context, falls back to legacy report parsing where possible, builds the main title bar text, switches the main tree into static report mode, and returns to live readings on request. | Lets support users inspect another machine's report through the normal Sensor Readout UI without manually extracting zipped reports first. |
| `SensorReadoutForm.ReportTools.cs` | Report comparison, privacy, and support flow | Adds two-report comparison, anonymized report saving, command-line report comparison/anonymized report helpers, and guided support report creation from existing diagnostic ZIP output. | Reuses the embedded report snapshot format so before/after support work stays aligned with normal reports. |
| `SensorReadoutForm.ReportsAndLogging.cs` | Reports and logging | Writes TXT/HTML reports, formats report output, opens save dialogs, manages log file paths, writes diagnostics, and includes HTML escaping/report helpers. | TXT and HTML reports show the Sensor Readout version near the top for support identification. TXT reports stay human-readable. HTML reports include the hidden structured snapshot used for exact report reopening. Logging is optional and stored under `Logs` when enabled. |
| `SensorReadoutForm.CoreTemp.cs` | Core Temp sensor import | Reads Core Temp shared memory and converts CPU temperature/load data into normal `SensorRow` objects. | Optional fallback path for systems where LibreHardwareMonitor does not expose CPU temperatures. |
| `SensorReadoutForm.GpuDetails.cs` | GPU registry helpers | Reads display-adapter BIOS and memory-size fields from the Windows display registry where available. | Shared by Display, Overview, and Performance GPU rows. |
| `SensorReadoutForm.HardwareDetails.cs` | Hardware detail collection and decoding | Collects and formats deeper Windows/WMI details for CPU, cache, memory arrays/modules, Windows, firmware, TPM, baseboard, chassis, and expansion slots. | Keeps verbose detail fields behind Details instead of cluttering the main tree. |
| `SensorReadoutForm.LibreHardwareMonitor.cs` | LibreHardwareMonitor collection | Opens and refreshes LibreHardwareMonitor, collects temperature/fan/control/storage rows, maintains fast/slow row cache behavior, returns cached rows when a previous LibreHardwareMonitor refresh is still running, and formats LibreHardwareMonitor storage/control names. | Main low-level hardware sensor source and writable fan-control source. |
| `SensorReadoutForm.Network.cs` | Network adapter readings | Reads .NET network-interface counters, IP/MAC/OUI details, and cached WMI adapter/configuration details. | Wi-Fi radio details are added by `SensorReadoutForm.Wifi.cs`; WMI-heavy details are cached so frequent refreshes do not keep `WmiPrvSE.exe` busy. |
| `SensorReadoutForm.Onboarding.cs` | First-run setup, tips, and settings reset | Shows the initial setup wizard for global hotkeys, the Help-menu hints and tips dialog, optional startup tips, and the Preferences-triggered delete-all-settings restart flow. | Keeps onboarding and recovery UI out of the main form constructor and Preferences implementation. |
| `SensorReadoutForm.Overview.cs` | Performance/Overview inventory rows | Builds Windows, firmware, motherboard, graphics summary, and system uptime overview rows. | Printer overview rows are added by `SensorReadoutForm.Printers.cs`; data-source rows are added by `SensorReadoutForm.TrendsAndSources.cs`. |
| `SensorReadoutForm.Performance.cs` | Live performance rows | Builds CPU, memory, GPU memory, GPU engine, and performance-counter rows for the Performance category. | Keeps fast live refresh logic separate from heavier WMI inventory/detail code. |
| `SensorReadoutForm.RegistryFormatting.cs` | Registry value formatting | Converts registry values, binary strings, and DWORD/QWORD-like data into readable text. | Shared by display, device, USB, audio, printer, and GPU detail code. |
| `SensorReadoutForm.Storage.cs` | SMART, disk, and storage topology | Collects Windows SMART, physical/logical disk, direct NVMe SMART fallbacks, logical disk performance counters, and disk/partition/volume Details. | Includes fixed and ready removable logical drives, Windows Storage physical-disk fields, and direct NVMe health-log fallbacks for USB/NVMe bridge cases where Windows permits access. |
| `SensorReadoutForm.WmiHelpers.cs` | Shared WMI helpers | Provides WMI value extraction, WMI date parsing, firmware/Secure Boot helpers, numeric conversion helpers, and common WMI text cleanup. | Used by device, display, printer, storage, overview, and detail collectors. |
| `SensorReadoutForm.SelfTest.cs` | Internal build-time self-tests | Runs the non-interactive `--self-test` workflow used by `Build.ps1 -SelfTest`, exercising settings persistence, collection, tree navigation, expansion state, tray text, quick tray/spoken hotkey assignments, alarms, fan curves, reports, ZIP report selection, static-report speech wording, diagnostics ZIP creation, and language/manual presence. | Intended for release and development checks on a disposable app copy; not a user-facing diagnostic feature. |
| `SensorReadoutForm.SettingsTransfer.cs` | Settings transfer | Exports and imports selective settings/profile packages, shows category checklists, resolves portable reading keys against the current machine, and imports hardware-bound items without active bindings. | Used by File > Export settings and profiles and File > Import settings and profiles. |
| `SensorReadoutForm.SettingsAndFiles.cs` | Settings and folders | Loads/saves per-computer JSON settings, migrates old top-level config/log files into `Config` and `Logs`, resolves app folders case-insensitively, backs up and removes nested duplicate update folders, and manages hidden reading persistence. | Keeps portable folder organization and backward compatibility together. Recovered stale update folders are zipped under `Backups`. |
| `SensorReadoutForm.StartupAndPrerequisites.cs` | Startup helpers and prerequisites | Checks/installs prerequisites, detects PawnIO, launches installer flow, offers optional Core Temp support, manages admin/elevation-related startup checks, creates/removes the Windows logon scheduled task used for Run at startup, cleans up legacy startup shortcuts/Run-key entries, and schedules first-refresh behavior. | Supports first-run and missing-driver guidance. The startup task runs with highest privileges so installed copies can start at logon despite the app's administrator manifest. |
| `SensorReadoutForm.TrendsAndSources.cs` | Reading history and data sources | Adds the Performance/Overview data-source summary rows and optional selected-reading CSV history logging. | Makes source transparency visible in the app and gives temporary troubleshooting trends without adding a database or heavy background monitor. |
| `SensorReadoutForm.Updates.cs` | Update checking and installer flow | Checks GitHub Releases, parses update metadata, shows update-available dialog, supports quiet update installs, handles the optional close/install/reopen confirmation, formats release notes, downloads ZIP assets, and runs the self-update script. | The updater replaces shipped folders cleanly instead of merging them, zips suspect or user-modified content under top-level `Backups`, and never copies Config, Logs, or Reports from an update ZIP over an existing installation. Used by Help > Check for updates and optional startup update checks. |
| `SensorReadoutForm.Usb.cs` | USB inventory and details | Collects USB devices, hubs, controllers, speeds, power where exposed, drive letters, safe-to-unplug status, USB network MAC/OUI data, storage hardware IDs, and detailed copyable fields. | Uses Windows device information plus the bundled `usb.ids` and OUI databases; also hosts the shared Details dialog as a grouped tree so raw WMI/registry fields do not interrupt practical fields. |
| `SensorReadoutForm.WmiRecovery.cs` | WMI recovery guard | Probes core WMI before slow Windows inventory refreshes and, when running elevated, makes one guarded `winmgmt` restart attempt for the specific stuck-provider failure that reports the paging file is too small. | Avoids silently leaving USB, Devices, Battery, and other WMI-backed sections blank after Windows WMI wedges, while refusing to restart WMI repeatedly or for unrelated missing-data cases. |
| `SensorReadoutForm.Wifi.cs` | Wi-Fi radio details | Uses the Windows WLAN API to collect connected Wi-Fi details such as SSID, access point, signal percentage, RSSI, channel, frequency, radio type, link speeds, and security fields. | Rows are merged into the normal Network category and can be selected for tray or spoken hotkey output. |
| `UsbIdDatabase.cs` | USB vendor/product lookup | Loads the bundled `usb.ids` file and resolves USB VID/PID pairs into friendlier vendor and product names. | Used by the USB category and USB detail view. |
| `PlugInZipImporter.cs` | Plug-in ZIP import | Validates imported plug-in ZIP files, finds the manifest, checks the plug-in id, prompts before replacing an existing plug-in folder, and copies the plug-in into `Plug-Ins`. | Imported plug-ins stay disabled until the user enables them in Preferences. |

| `PlugIns\LenovoThinkPad\plugin.json` | Lenovo manifest | Describes the Lenovo Laptop plug-in and declares Help-menu support links for Lenovo Vantage and Lenovo System Update. | Manifest links appear only when the plug-in is enabled. |

| `PlugIns\MsiLaptop\plugin.json` | MSI manifest | Describes the MSI Laptop plug-in and declares Help-menu support links for MSI Center and MSI Support. | Manifest links appear only when the plug-in is enabled. |

On compatible MSI models it exposes CPU/GPU MSI fan-table controls; manual writes capture the original table/mode first, and automatic/default restores the original state when available. |
