Sensor Readout

Versione attuale: 1.2.1.

Sensor Readout e un'utilita Windows per leggere sensori hardware e controllare ventole compatibili, con interfaccia pensata prima di tutto per tastiera e screen reader.

A sinistra mostra categorie principali, a destra una vista ad albero con letture raggruppate per dispositivo o scopo, e i comandi comuni sono in una normale barra menu.

Sensor Readout e disponibile su GitHub.

Cosa fa

  • Legge temperature, RPM ventole, salute e capacita storage, e contatori hardware selezionati.
  • Mostra Prestazioni/Panoramica con uptime, BIOS, GPU, uso CPU, uso memoria e attivita lettura/scrittura delle unita.
  • Apre subito l'interfaccia principale mentre il primo aggiornamento sensori continua in background.
  • Mostra rete con stato adattatore, indirizzi IP, velocita collegamento, invio/ricezione e traffico totale.
  • Collega la pagina progetto da README, menu Aiuto e finestra Informazioni.
  • Usa librerie LibreHardwareMonitor incluse per l'accesso ai sensori.
  • Usa il driver PawnIO per sensori scheda madre e controllo ventole dove l'hardware lo supporta.
  • Permette nomi facili per intestazioni ventola.
  • Nasconde per impostazione predefinita intestazioni ferme o non popolate.
  • Applica percentuali manuali a una ventola selezionata o a tutte le ventole visibili.
  • Riporta una o tutte le ventole al controllo automatico/predefinito.
  • Salva report TXT o HTML.
  • Supporta aggiornamento automatico configurabile e usa 5 secondi come intervallo iniziale.
  • Puo avviarsi con Windows e partire minimizzato nell'area di notifica.
  • Usa un file di configurazione per computer in Config, ad esempio Desktop.json o Laptop.json, cosi una cartella Dropbox o USB funziona su piu macchine senza sovrascrivere impostazioni.
  • Salva subito le modifiche alle preferenze, proteggendo hotkey, area di notifica, elementi nascosti e opzioni simili.
  • Scrive diagnostica in Logs\<ComputerName>.log quando il logging e attivo.
  • Mostra temperature in Celsius o Fahrenheit.
  • Converte contatori SMART Samsung in valori GB/TB leggibili.
  • Supporta hotkey globali opzionali per mostrare/nascondere e leggere lo stato dell'area di notifica.
  • Puo parlare tramite NVDA usando le DLL NVDA Controller Client a 64 bit incluse.
  • Supporta file lingua modificabili nella cartella langs.
  • Il logging e disattivato per impostazione predefinita e si puo abilitare per diagnosi.
  • Puo mostrare letture scelte nel tooltip dell'area di notifica.

Requisiti

  • Windows 10 o Windows 11, 64 bit.
  • Microsoft .NET Framework 4.8 o successivo.
  • Driver PawnIO installato per sensori scheda madre e controllo ventole.
  • Diritti amministratore per leggere molti sensori scheda madre o controllare ventole.

LibreHardwareMonitor non deve essere eseguito come app separata perche questa cartella include le librerie necessarie. L'app autonoma puo essere utile per diagnosi.

Comandi winget

Aprire Windows Terminal o PowerShell come amministratore ed eseguire:

winget install --id Microsoft.DotNet.Framework.Runtime -e --source winget
winget install --id namazso.PawnIO -e --source winget

Strumento opzionale di diagnosi:

winget install --id LibreHardwareMonitor.LibreHardwareMonitor -e --source winget

Install-Prerequisites.cmd richiama Install-Prerequisites.ps1, chiede amministratore se serve, controlla winget e installa .NET Framework Runtime e PawnIO. Per installare anche LibreHardwareMonitor autonomo:

.\Install-Prerequisites.ps1 -IncludeLibreHardwareMonitor

Primi passi

1. Avviare Sensor Readout.exe.

2. Accettare la richiesta amministratore di Windows.

3. Se PawnIO manca, Sensor Readout propone l'installatore dei requisiti.

4. Premere F5 se i valori stanno ancora caricando.

5. Aprire Opzioni > Controllo ventole... o premere Ctrl+L.

Se mancano ventole o controlli scheda madre, controllare PawnIO e l'avvio come amministratore. L'installatore e disponibile piu tardi da Aiuto > Installa requisiti....

Preferenze

Usare Opzioni > Preferenze o Ctrl+,.

  • Aggiornamento automatico.
  • Aggiornamento mentre Sensor Readout ha il focus.
  • Intervallo di aggiornamento.
  • Icona area di notifica.
  • Avvio con Windows e avvio minimizzato.
  • Fino a quattro letture nel tooltip dell'area di notifica.
  • Letture e gruppi nascosti.
  • Livello logging: Disattivato, Errore, Normale o Debug.

Quando lo stato area di notifica e attivo, minimizzare rimuove Sensor Readout da barra applicazioni e Alt+Tab. Riaprirlo dall'icona. Alt+F4 chiude l'app.

L'opzione di avvio crea o rimuove Sensor Readout.lnk nella cartella Startup dell'utente e abilita l'avvio minimizzato.

Le letture area di notifica passano da una lista disponibile a un ordine. Il limite e quattro perche i tooltip Windows sono brevi. Una lettura appare in una sola lista. Usare pulsanti o Ctrl+frecce. Gli elementi mostrano dispositivo, nome e categoria, ad esempio Ethernet - Rx: Rete.

Lettura sensori

Il pannello letture e una vista ad albero. Temperature, Ventole, SMART, Prestazioni e Rete raggruppano per dispositivo o scopo e poi per valore.

La lista sinistra cambia solo vista; non abilita ne disabilita dispositivi. Sensor Readout parte su Temperature. Prestazioni raggruppa CPU, memoria e attivita disco. Rete mostra stato, IP, velocita, tassi e contatori.

Le percentuali sono esposte anche nella barra di avanzamento inferiore per gli screen reader. Usare Modifica, tasto Applicazioni o clic destro per copiare, rinominare una ventola o nascondere valori.

Flusso controllo ventole

Aprire Opzioni > Controllo ventole... o Ctrl+L.

1. Scegliere un controllo. 2. Opzionale: dare un nome e premere Alt+L. 3. Inserire percentuale 0-100. 4. Alt+M applica. 5. Alt+A riporta la selezione ad automatico. 6. Alt+R riporta tutti i visibili ad automatico.

I pulsanti globali agiscono solo sui controlli visibili. Intestazioni ferme o vuote sono nascoste finche Mostra ferme non e attivo. I nomi sono salvati in Config e cambiano solo l'aspetto in Sensor Readout.

Report

Premere Ctrl+S o usare File > Salva report.... TXT raggruppa per tipo e dispositivo. HTML usa tabelle con sensore, valore e fonte.

File

Build portable: Sensor Readout.exe. Sorgente: src\SensorReadoutApp.cs e manifest.

  • Config\<ComputerName>.json: preferenze aggiornamento, area notifica, elementi nascosti, nomi ventola e logging.
  • Logs\<ComputerName>.log: diagnostica opzionale, azioni ventola, hotkey e messaggi NVDA.

File lingua

  • Mettere file .txt o .ini in langs.
  • Al primo avvio Sensor Readout sceglie la lingua Windows se presente, altrimenti inglese.
  • Usare key=value; # e ; sono commenti.
  • language.name e il nome visualizzato.
  • manual.file=README-xx.html indica il manuale per F1. Solo nome file, senza percorso. Cerca in docs, poi usa inglese.
  • number.decimalSeparator=, imposta il separatore decimale predefinito.
  • La cartella viene controllata circa ogni 15 secondi.
  • langs\English.txt e lingua principale e modello.
  • Il separatore decimale puo essere cambiato anche in Preferenze.

La voce NVDA opzionale usa nvdaControllerClient.dll e nvdaControllerClient64.dll se NVDA e in esecuzione. All'avvio minimizzato puo parlare un messaggio configurabile. L'editor lingua consente modifiche senza altro editor.

Note

Il supporto ventole dipende da LibreHardwareMonitor, PawnIO, chip sensore e BIOS. Alcuni sistemi danno letture ma non controlli scrivibili. L'accesso Super I/O scheda madre spesso richiede amministratore.
